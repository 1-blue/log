## 🤔 프로토타입 체인, 왜 중요할까요?

많은 개발자분들이 JavaScript의 객체 지향적 특성을 이해하는 데 어려움을 겪으시곤 해요.<br />
클래스 기반 언어에 익숙한 분들에게는 JavaScript의 "프로토타입 기반" 상속이 낯설게 느껴질 수 있거든요.<br />
하지만 이 메커니즘을 이해하는 것은 JavaScript 마스터의 필수 관문이라고 할 수 있어요.

### 0️⃣ 자바스크립트 객체 상속의 미스터리

JavaScript에서 객체를 생성하고 특정 속성이나 메서드를 호출할 때, 해당 객체에 직접 속성이나 메서드가 없더라도 정상적으로 동작하는 경우가 많아요.<br />
예를 들어, 배열 인스턴스가 `map()`이나 `forEach()`와 같은 메서드를 사용할 수 있는 것은 바로 프로토타입 체인 덕분이에요.<br />
이러한 동작의 배경에는 JavaScript의 독특한 상속 방식인 "프로토타입 체인"이 숨어있답니다.<br />

```tsx showLineNumbers
const arr = [1, 2, 3];
console.log(arr.hasOwnProperty('length')); // true
console.log(arr.hasOwnProperty('map'));    // false
console.log(arr.map);                      // f map() { [native code] }
```

위 코드에서 `arr` 객체는 `map` 메서드를 직접 가지고 있지 않지만, `arr.map`을 호출하면 오류 없이 `map` 함수를 찾아서 실행해요.<br />
`hasOwnProperty`는 `arr` 객체가 직접 `length` 속성을 가지고 있는지는 알려주지만, `map` 메서드는 가지고 있지 않다고 하죠.<br />
그렇다면 `map`은 어디에서 온 걸까요? 바로 프로토타입 체인을 통해 상속받은 것이랍니다.

### 1️⃣ `__proto__`와 `prototype`의 혼란

JavaScript를 공부하다 보면 `__proto__`와 `prototype`이라는 두 가지 용어를 자주 접하게 되는데요.<br />
이 두 가지는 이름이 비슷해서 혼동하기 쉽지만, 실제로는 완전히 다른 역할을 하는 속성들이에요.<br />
이들의 차이를 명확히 이해하는 것이 프로토타입 체인을 이해하는 첫걸음이 된답니다.

## ⚙️ 프로토타입 체인의 핵심 원리 파헤치기

프로토타입 체인은 간단히 말해, 객체들이 다른 객체로부터 속성과 메서드를 상속받기 위해 연결된 링크들의 연속이라고 볼 수 있어요.<br />
어떤 객체의 속성이나 메서드를 찾을 때, 해당 객체에 없으면 연결된 상위 객체(프로토타입)에서 찾고, 거기에도 없으면 또 그 상위 프로토타입에서 찾는 과정을 반복하는 것이죠.

### 0️⃣ `[[Prototype]]` 내부 슬롯과 `__proto__` 접근자

모든 JavaScript 객체는 `[[Prototype]]`이라는 내부 슬롯을 가지고 있어요.<br />
이 `[[Prototype]]`은 해당 객체의 부모 역할을 하는 프로토타입 객체를 가리키는 숨겨진 참조라고 생각하시면 돼요.<br />
우리는 이 `[[Prototype]]`에 직접 접근할 수는 없지만, 대부분의 환경에서 `__proto__`라는 특별한 접근자 속성을 통해 간접적으로 접근할 수 있어요.<br />

<Blockquote type="warning">
`__proto__`는 표준화된 속성이 아니었지만(ES6에서 비로소 표준화되었어요), 여전히 사용을 권장하지 않아요.<br />
성능 문제나 예상치 못한 부작용을 일으킬 수 있어서, 대신 `Object.getPrototypeOf()`나 `Object.setPrototypeOf()` 같은 표준 API를 사용하는 것이 좋아요.
</Blockquote>

```tsx showLineNumbers
const person = {
  name: '블루',
  greet() {
    console.log(`안녕하세요, ${this.name}이에요.`);
  }
};

const developer = {
  __proto__: person, // developer의 프로토타입을 person으로 설정
  skill: 'JavaScript',
  code() {
    console.log(`${this.skill} 코드를 작성하고 있어요.`);
  }
};

developer.greet(); // 안녕하세요, 블루이에요. (person으로부터 상속받음)
developer.code();  // JavaScript 코드를 작성하고 있어요.
console.log(developer.name); // 블루 (person으로부터 상속받음)
```

위 예시에서 `developer` 객체는 `greet` 메서드나 `name` 속성을 직접 가지고 있지 않지만, `__proto__`를 통해 `person` 객체를 프로토타입으로 설정했기 때문에 `person`의 속성과 메서드를 마치 자신의 것처럼 사용할 수 있어요.

### 1️⃣ `prototype` 속성과 생성자 함수

`prototype` 속성은 오직 **함수 객체**에만 존재하는 특별한 속성이에요.<br />
이 `prototype` 속성은 해당 함수가 `new` 키워드와 함께 생성자 함수로 호출될 때, 새로 생성되는 객체의 `[[Prototype]]`이 될 객체를 가리켜요.<br />
즉, 생성자 함수를 통해 만들어질 인스턴스들이 공유할 속성이나 메서드들을 정의하는 공간이라고 생각하시면 돼요.

```tsx showLineNumbers
function User(name) {
  this.name = name;
}

// User.prototype에 greet 메서드 추가
User.prototype.greet = function() {
  console.log(`안녕하세요, 제 이름은 ${this.name}이에요.`);
};

const user1 = new User('앨리스');
const user2 = new User('밥');

user1.greet(); // 안녕하세요, 제 이름은 앨리스이에요.
user2.greet(); // 안녕하세요, 제 이름은 밥이에요.

console.log(user1.__proto__ === User.prototype); // true
console.log(user2.__proto__ === User.prototype); // true
console.log(user1.greet === user2.greet);       // true (같은 함수를 공유)
```

이 예시에서 `User` 함수는 생성자 함수로 사용되고 있어요.<br />
`User.prototype`에 `greet` 메서드를 추가하면, `new User()`로 생성된 모든 `user1`, `user2` 인스턴스들이 이 `greet` 메서드를 공유하게 돼요.<br />
각 인스턴스는 `greet` 메서드를 직접 가지고 있지 않고, `[[Prototype]]`을 통해 `User.prototype`에 있는 `greet` 메서드를 찾아 사용하는 것이죠.<br />
이것은 메모리 측면에서 매우 효율적인 방법이에요. 각 인스턴스마다 같은 메서드를 복사할 필요 없이, 한 번만 정의해두고 공유할 수 있으니까요.

### 2️⃣ 체인 동작 방식: 속성 탐색 과정

JavaScript 엔진이 객체의 속성이나 메서드를 찾을 때 프로토타입 체인은 다음과 같은 순서로 동작해요.<br />

<Blockquote type="info">
**속성 탐색 과정**<br />
1.  **객체 자신 확인**: 먼저 해당 속성이나 메서드가 객체 자신에게 직접 존재하는지 확인해요.<br />
2.  **프로토타입 확인**: 객체 자신에게 없으면, 객체의 `[[Prototype]]`이 가리키는 프로토타입 객체로 이동해서 해당 속성을 찾아요.<br />
3.  **체인 상위로 이동**: 프로토타입 객체에도 없으면, 그 프로토타입 객체의 `[[Prototype]]`으로 또 이동해서 찾아요.<br />
4.  **`null` 도달**: 이 과정을 반복하다가 `[[Prototype]]`이 `null`인 객체(보통 `Object.prototype`의 `[[Prototype]]`이 `null`이에요)에 도달할 때까지 계속 탐색해요.<br />
5.  **`undefined` 반환**: `null`에 도달했는데도 속성을 찾지 못하면 `undefined`를 반환한답니다.
</Blockquote>

```tsx showLineNumbers
function Animal(name) {
  this.name = name;
}

Animal.prototype.speak = function() {
  console.log(`${this.name}이(가) 소리를 내요.`);
};

function Dog(name, breed) {
  Animal.call(this, name); // Animal 생성자 호출하여 name 속성 상속
  this.breed = breed;
}

// Dog의 프로토타입을 Animal의 프로토타입으로 연결
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog; // constructor 속성 복구

Dog.prototype.bark = function() {
  console.log(`${this.name}이(가) 멍멍 짖어요!`);
};

const myDog = new Dog('바둑이', '진돗개');

myDog.bark();  // 바둑이이(가) 멍멍 짖어요! (Dog.prototype)
myDog.speak(); // 바둑이이(가) 소리를 내요. (Animal.prototype)
console.log(myDog.name); // 바둑이 (myDog 자신)
```

위 예시에서 `myDog.speak()`를 호출하면 다음과 같은 순서로 탐색이 이루어져요.<br />
1.  `myDog` 객체에 `speak` 메서드가 있는지 확인해요. (없음)<br />
2.  `myDog`의 `[[Prototype]]`인 `Dog.prototype`으로 이동해서 `speak` 메서드를 확인해요. (없음)<br />
3.  `Dog.prototype`의 `[[Prototype]]`인 `Animal.prototype`으로 이동해서 `speak` 메서드를 확인해요. (여기서 찾음!)<br />
4.  `Animal.prototype`의 `speak` 메서드를 실행해요.

이것이 바로 프로토타입 체인의 동작 방식이에요.

## 🛠️ 실전! 프로토타입 체인 활용 패턴

프로토타입 체인을 이해했다면, 이제 이를 활용하여 객체 지향적인 코드를 작성하는 다양한 방법을 알아볼까요?

### 0️⃣ 생성자 함수를 이용한 상속

가장 전통적인 JavaScript의 상속 패턴이에요.<br />
생성자 함수를 통해 인스턴스의 속성을 초기화하고, `prototype` 속성에 메서드를 추가하여 상속을 구현해요.

```tsx showLineNumbers
function Vehicle(type) {
  this.type = type;
}

Vehicle.prototype.drive = function() {
  console.log(`${this.type}이(가) 움직여요.`);
};

function Car(brand) {
  Vehicle.call(this, '자동차'); // 부모 생성자 호출
  this.brand = brand;
}

// Car의 프로토타입을 Vehicle.prototype을 상속받도록 설정
Car.prototype = Object.create(Vehicle.prototype);
Car.prototype.constructor = Car; // constructor 속성 재설정

Car.prototype.honk = function() {
  console.log(`${this.brand} 자동차가 빵빵거려요!`);
};

const myCar = new Car('현대');
myCar.drive(); // 자동차이(가) 움직여요.
myCar.honk();  // 현대 자동차가 빵빵거려요!
```

여기서 `Object.create(Vehicle.prototype)`은 `Vehicle.prototype`을 프로토타입으로 하는 새로운 객체를 생성하고, 이를 `Car.prototype`으로 할당하는 핵심적인 부분이에요.<br />
이렇게 함으로써 `Car` 인스턴스는 `Vehicle.prototype`에 정의된 `drive` 메서드를 상속받을 수 있게 된답니다.<br />
`constructor` 속성을 재설정하는 것도 중요한데요, `Object.create()`를 사용하면 `constructor`가 부모의 것으로 설정되기 때문에, 올바른 생성자를 가리키도록 다시 지정해 주어야 해요.

### 1️⃣ `Object.create()`를 이용한 명시적 상속

`Object.create()` 메서드는 지정된 프로토타입 객체를 가지는 새로운 객체를 생성해요.<br />
이는 특정 객체를 직접적인 부모로 삼아 새로운 객체를 만들고 싶을 때 매우 유용하게 사용돼요.

<LinkPreviewCard href="https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Object/create" />

```tsx showLineNumbers
const personPrototype = {
  greet() {
    console.log(`안녕하세요, 제 이름은 ${this.name}이에요.`);
  }
};

const john = Object.create(personPrototype); // personPrototype을 프로토타입으로 하는 객체 생성
john.name = '존';
john.greet(); // 안녕하세요, 제 이름은 존이에요.

const jane = Object.create(personPrototype, {
  name: { value: '제인', writable: true, configurable: true, enumerable: true }
});
jane.greet(); // 안녕하세요, 제 이름은 제인이에요.
```

`Object.create()`는 두 번째 인자로 속성 디스크립터를 받을 수도 있어서, 객체 생성 시 속성의 특성을 세밀하게 제어할 수 있어요.<br />
이 방식은 상속 체인을 명시적으로 설정할 때 유용하며, 특히 믹스인(Mixin) 패턴을 구현할 때도 활용될 수 있답니다.

### 2️⃣ 클래스(ES6)와 프로토타입 체인

ES6에서 도입된 `class` 문법은 JavaScript에 클래스 기반 객체 지향 프로그래밍의 문법을 제공하지만, 내부적으로는 여전히 프로토타입 체인을 기반으로 동작하는 "문법적 설탕(Syntactic Sugar)"이에요.<br />
즉, `class` 키워드를 사용하더라도 JavaScript의 객체 모델이 변경되는 것은 아니라는 점을 기억하는 것이 중요해요.

```tsx showLineNumbers
class Person {
  constructor(name) {
    this.name = name;
  }

  greet() {
    console.log(`안녕하세요, ${this.name}이에요.`);
  }
}

class Student extends Person {
  constructor(name, studentId) {
    super(name); // 부모 클래스 생성자 호출
    this.studentId = studentId;
  }

  study() {
    console.log(`${this.name} 학생이 공부하고 있어요. (학번: ${this.studentId})`);
  }
}

const student = new Student('철수', '20230001');
student.greet(); // 안녕하세요, 철수이에요. (Person 클래스의 메서드 상속)
student.study(); // 철수 학생이 공부하고 있어요. (학번: 20230001)

console.log(student instanceof Student); // true
console.log(student instanceof Person);  // true
console.log(Object.getPrototypeOf(student) === Student.prototype); // true
console.log(Object.getPrototypeOf(Student.prototype) === Person.prototype); // true
```

`extends` 키워드를 사용하면 `Student.prototype`의 `[[Prototype]]`이 자동으로 `Person.prototype`을 가리키도록 설정돼요.<br />
`super()` 호출은 부모 클래스의 생성자를 호출하여 `this` 컨텍스트를 올바르게 초기화하는 역할을 한답니다.<br />
결과적으로 클래스를 사용하더라도, 모든 상속은 프로토타입 체인을 통해 이루어진다는 것을 알 수 있어요.

### 3️⃣ 성능 최적화와 메모리 효율

프로토타입 체인을 이해하는 것은 코드의 성능과 메모리 효율성을 높이는 데도 중요해요.<br />
메서드를 각 인스턴스에 직접 정의하는 대신 프로토타입에 정의하면, 모든 인스턴스가 하나의 메서드 함수를 공유하게 되어 메모리 사용량을 크게 줄일 수 있어요.<br />

<Blockquote type="tip">
**메모리 효율적인 코드 작성 팁**<br />
*   **메서드는 프로토타입에**: 생성자 함수의 `prototype`에 메서드를 정의하여 모든 인스턴스가 공유하도록 하세요.<br />
*   **속성은 인스턴스에**: 각 인스턴스마다 고유해야 하는 데이터(예: `name`, `id`)는 생성자 함수 내부에서 `this`를 통해 초기화하세요.
</Blockquote>

```diff-tsx showLineNumbers
// ❌ 비효율적인 방식: 매번 새로운 함수 생성
function BadUser(name) {
  this.name = name;
-  this.greet = function() {
-    console.log(`안녕하세요, ${this.name}이에요.`);
-  };
}

// ✅ 효율적인 방식: 프로토타입에 함수 공유
function GoodUser(name) {
  this.name = name;
}
+
+GoodUser.prototype.greet = function() {
+  console.log(`안녕하세요, ${this.name}이에요.`);
+};

const badUser1 = new BadUser('앨리스');
const badUser2 = new BadUser('밥');
console.log(badUser1.greet === badUser2.greet); // false (서로 다른 함수 객체)

const goodUser1 = new GoodUser('찰리');
const goodUser2 = new GoodUser('데이비드');
console.log(goodUser1.greet === goodUser2.greet); // true (같은 함수 객체 공유)
```

위 `diff` 코드를 보면, `BadUser` 방식은 인스턴스가 생성될 때마다 새로운 `greet` 함수가 만들어져서 메모리 낭비가 발생해요.<br />
반면 `GoodUser` 방식은 `greet` 함수가 `GoodUser.prototype`에 한 번만 정의되고, 모든 인스턴스가 이를 공유하므로 훨씬 효율적이랍니다.

## 🚨 주의할 점과 흔한 오해

프로토타입 체인을 사용할 때 몇 가지 주의할 점이 있어요.

### 0️⃣ `__proto__` 직접 접근 피하기

앞서 언급했듯이 `__proto__`는 표준화된 접근자이지만, 여전히 직접적인 사용은 권장되지 않아요.<br />
대신 `Object.getPrototypeOf()`와 `Object.setPrototypeOf()` 같은 표준 메서드를 사용해야 해요.<br />
이러한 메서드는 더 안전하고 예측 가능한 방식으로 프로토타입에 접근하고 변경할 수 있도록 도와준답니다.

<LinkPreviewCard href="https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Object/getprototypeof" />
<LinkPreviewCard href="https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Object/setprototypeof" />

```tsx showLineNumbers
const obj = {};
const proto = { value: 42 };

// ❌ 권장하지 않음
// obj.__proto__ = proto;

// ✅ 권장하는 방식: Object.setPrototypeOf()
Object.setPrototypeOf(obj, proto);

console.log(obj.value); // 42
console.log(Object.getPrototypeOf(obj) === proto); // true
```

### 1️⃣ `instanceof` 연산자의 작동 원리

`instanceof` 연산자는 객체가 특정 생성자 함수의 인스턴스인지 확인하는 데 사용돼요.<br />
이 연산자는 내부적으로 **오른쪽 피연산자(생성자 함수)의 `prototype`이 왼쪽 피연산자(객체)의 프로토타입 체인 어딘가에 존재하는지**를 확인해요.<br />

<Blockquote type="tip">
`instanceof`는 객체의 실제 생성자를 확인하는 것이 아니라, 프로토타입 체인에 특정 `prototype` 객체가 존재하는지 확인하는 것이에요.<br />
따라서 프로토타입 체인을 수동으로 변경하면 `instanceof` 결과도 달라질 수 있어요.
</Blockquote>

```tsx showLineNumbers
function MyObject() {}
const instance = new MyObject();

console.log(instance instanceof MyObject); // true
console.log(instance instanceof Object);   // true (MyObject.prototype의 프로토타입은 Object.prototype)

const anotherObject = {};
console.log(anotherObject instanceof MyObject); // false
console.log(anotherObject instanceof Object);   // true
```

## 📝 정리하며

오늘은 JavaScript의 핵심 개념인 프로토타입 체인에 대해 심층적으로 알아보았어요.<br />
`[[Prototype]]`, `__proto__`, `prototype`의 차이점을 명확히 이해하고, 프로토타입 체인이 어떻게 동작하는지, 그리고 이를 실무에서 어떻게 활용하고 주의해야 하는지에 대해 다루었답니다.

### 0️⃣ 핵심 요약

*   **프로토타입 체인**: 객체가 속성이나 메서드를 찾을 때, 자신에게 없으면 부모 프로토타입으로 거슬러 올라가 탐색하는 메커니즘이에요.<br />
*   **`[[Prototype]]`**: 모든 객체가 가지는 내부 슬롯으로, 부모 프로토타입 객체를 가리켜요.<br />
*   **`__proto__`**: `[[Prototype]]`에 간접적으로 접근하는 접근자 속성이지만, 직접적인 사용은 권장되지 않아요.<br />
*   **`prototype`**: 오직 함수 객체에만 존재하며, 해당 함수가 생성자로 사용될 때 생성될 인스턴스들의 `[[Prototype]]`이 될 객체를 가리켜요.<br />
*   **클래스(ES6)**: 프로토타입 체인의 문법적 설탕이며, 내부적으로는 여전히 프로토타입 기반 상속을 사용해요.<br />
*   **활용**: 생성자 함수, `Object.create()`, ES6 클래스를 통해 효율적인 객체 상속 및 메모리 관리를 할 수 있어요.

### 1️⃣ 다음 단계로 나아가기

프로토타입 체인을 이해하는 것은 JavaScript의 객체 지향 프로그래밍을 마스터하는 데 필수적인 단계예요.<br />
이 지식을 바탕으로 여러분의 코드를 더욱 견고하고 효율적으로 만들어나가시길 바라요.<br />
