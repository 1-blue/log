## 🚀 React 커스텀 훅, 왜 중요할까요?

React 훅(Hooks)이 도입되면서 함수형 컴포넌트에서 상태 관리와 생명 주기 로직을 훨씬 깔끔하게 다룰 수 있게 되었어요. 특히 커스텀 훅은 컴포넌트 간에 **상태 로직을 재사용**할 수 있도록 해주어 개발 효율을 크게 높여주는 강력한 도구예요.

### 0️⃣ 컴포넌트 로직의 재사용성

여러 컴포넌트에서 동일하거나 유사한 로직(예: 데이터 페칭, 폼 입력 처리, 브라우저 API 연동)을 사용해야 할 때가 많아요. 커스텀 훅은 이런 로직을 한곳에 모아 추상화함으로써, 중복 코드를 줄이고 필요한 곳에서 쉽게 가져다 쓸 수 있게 해줘요. 마치 유틸리티 함수처럼요.<br />
이는 코드 베이스의 DRY(Don't Repeat Yourself) 원칙을 지키는 데 큰 도움이 된답니다.

### 1️⃣ 관심사의 분리 (Separation of Concerns)

컴포넌트는 UI를 렌더링하는 역할에 집중하고, 비즈니스 로직이나 부수 효과(side effects) 처리는 커스텀 훅으로 분리할 수 있어요. 이렇게 관심사를 분리하면 컴포넌트의 책임이 명확해지고, 코드의 가독성과 유지보수성이 크게 향상돼요. 복잡한 컴포넌트도 훨씬 이해하기 쉬워지죠.

### 2️⃣ 복잡성 관리와 가독성 향상

만약 하나의 컴포넌트에 너무 많은 `useState`, `useEffect` 등의 훅이 사용된다면, 컴포넌트 코드가 길어지고 복잡해질 수 있어요. 커스텀 훅을 사용하면 이러한 로직들을 의미 있는 단위로 캡슐화하여, 컴포넌트 내부를 훨씬 깔끔하고 간결하게 유지할 수 있어요. 마치 작은 함수들을 조합해서 큰 프로그램을 만드는 것과 같다고 생각하시면 돼요.

## 💡 커스텀 훅 설계의 핵심 원칙

좋은 커스텀 훅을 만들기 위해서는 몇 가지 중요한 원칙들을 이해하고 적용해야 해요. 이 원칙들은 훅의 재사용성과 유지보수성을 결정하는 핵심 요소가 된답니다.

### 0️⃣ 단일 책임 원칙 (Single Responsibility Principle, SRP)

하나의 커스텀 훅은 하나의 명확한 책임만 가져야 해요. 예를 들어, 데이터 페칭과 폼 유효성 검사를 동시에 처리하는 훅보다는, `useFetch`와 `useFormValidation`처럼 각각의 책임을 분리하는 것이 좋아요. 이렇게 하면 훅의 목적이 명확해지고, 다른 곳에서 재사용하기도 훨씬 쉬워져요.<br />
훅이 너무 많은 일을 하려고 하면 복잡해지고, 테스트하기도 어려워진답니다.

### 1️⃣ 추상화 레벨 유지

커스텀 훅은 내부 구현 세부 사항을 숨기고, 명확한 인터페이스를 통해 필요한 기능만 노출해야 해요. 훅을 사용하는 컴포넌트는 훅의 내부 로직을 알 필요 없이, 단순히 훅이 제공하는 값과 함수만 활용할 수 있어야 해요. 이는 캡슐화의 중요한 개념이기도 해요.

### 2️⃣ 명확한 인터페이스 (Input/Output)

커스텀 훅은 어떤 인자를 받아서 어떤 값을 반환하는지 명확해야 해요. 인자는 훅의 동작을 설정하는 데 사용되고, 반환 값은 훅의 결과물이나 상태, 그리고 상태를 변경하는 함수가 될 수 있어요. 훅의 인터페이스가 직관적일수록 사용하기 편리해져요.

### 3️⃣ 상태 관리와 부수 효과

커스텀 훅은 `useState`, `useEffect`, `useRef`, `useCallback`, `useMemo` 등 React의 기본 훅들을 조합하여 상태를 관리하고 부수 효과를 처리해요. 훅 내부에서 이런 기본 훅들을 어떻게 효과적으로 사용하는지가 커스텀 훅의 성능과 안정성에 큰 영향을 미친답니다.<br />
특히 `useEffect`의 의존성 배열과 클린업 함수는 메모리 누수를 방지하고 불필요한 재실행을 막는 데 매우 중요해요.

## 🛠️ 실용적인 커스텀 훅 패턴과 예시

이제 몇 가지 일반적인 시나리오에 적용할 수 있는 커스텀 훅 패턴과 실제 코드 예시를 살펴볼게요. 이 예시들을 통해 위에서 설명한 설계 원칙들이 어떻게 적용되는지 확인하실 수 있을 거예요.

### 0️⃣ 데이터 페칭 훅 (useFetch 예시)

많은 React 애플리케이션에서 데이터를 가져오는 로직은 필수적이에요. `useFetch` 훅은 로딩 상태, 에러 처리, 그리고 데이터 자체를 추상화하여 제공할 수 있어요.

```tsx showLineNumbers
import { useState, useEffect } from 'react';

interface FetchResult<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
}

function useFetch<T>(url: string): FetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState<boolean>(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      setLoading(true);
      setError(null);
      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        if (err instanceof Error) {
          setError(err);
        } else {
          setError(new Error('An unknown error occurred'));
        }
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]); // url이 변경될 때마다 데이터를 다시 가져와요.

  return { data, loading, error };
}

// 사용 예시
function UserProfile({ userId }: { userId: number }) {
  const { data: user, loading, error } = useFetch<{ name: string; email: string }>(`https://jsonplaceholder.typicode.com/users/${userId}`);

  if (loading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {error.message}</div>;
  if (!user) return null;

  return (
    <div>
      <h2>{user.name}</h2>
      <p>이메일: {user.email}</p>
    </div>
  );
}
```

위 `useFetch` 훅은 URL을 인자로 받아 데이터, 로딩 상태, 에러를 반환해요. 컴포넌트는 이 훅을 사용하여 데이터 페칭 로직에 대한 걱정 없이 UI 렌더링에만 집중할 수 있게 되죠.

### 1️⃣ 폼 상태 관리 훅 (useForm 예시)

폼 입력 값과 유효성 검사를 관리하는 훅은 재사용성이 매우 높아요. `useForm` 훅은 여러 입력 필드의 상태를 한곳에서 관리하고, 제출 로직을 추상화할 수 있어요.

```tsx showLineNumbers
import { useState, useCallback } from 'react';

interface FormValues {
  [key: string]: string;
}

interface FormHooks {
  values: FormValues;
  handleChange: (event: React.ChangeEvent<HTMLInputElement | HTMLTextAreaElement>) => void;
  handleSubmit: (callback: (values: FormValues) => void) => (event: React.FormEvent) => void;
  resetForm: () => void;
}

function useForm(initialValues: FormValues = {}): FormHooks {
  const [values, setValues] = useState<FormValues>(initialValues);

  const handleChange = useCallback((event: React.ChangeEvent<HTMLInputElement | HTMLTextAreaElement>) => {
    // React 17 이전 버전과의 호환성을 위해 event.persist()를 사용하기도 했지만, 최신 React에서는 이벤트 객체가 풀링되지 않아 불필요해요.
    setValues(prevValues => ({
      ...prevValues,
      [event.target.name]: event.target.value,
    }));
  }, []);

  const handleSubmit = useCallback((callback: (values: FormValues) => void) => (event: React.FormEvent) => {
    event.preventDefault();
    callback(values);
  }, [values]); // values가 변경될 때마다 handleSubmit 함수를 재생성해요.

  const resetForm = useCallback(() => {
    setValues(initialValues);
  }, [initialValues]); // initialValues가 변경될 때마다 resetForm 함수를 재생성해요.

  return { values, handleChange, handleSubmit, resetForm };
}

// 사용 예시
function ContactForm() {
  const { values, handleChange, handleSubmit, resetForm } = useForm({
    name: '',
    email: '',
    message: '',
  });

  const onSubmit = (formValues: FormValues) => {
    console.log('폼 제출:', formValues);
    alert(`폼 제출됨!\n이름: ${formValues.name}\n이메일: ${formValues.email}\n메시지: ${formValues.message}`);
    resetForm();
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <div>
        <label htmlFor="name">이름:</label><br />
        <input type="text" id="name" name="name" value={values.name} onChange={handleChange} />
      </div>
      <div>
        <label htmlFor="email">이메일:</label><br />
        <input type="email" id="email" name="email" value={values.email} onChange={handleChange} />
      </div>
      <div>
        <label htmlFor="message">메시지:</label><br />
        <textarea id="message" name="message" value={values.message} onChange={handleChange}></textarea>
      </div>
      <button type="submit">제출</button>
      <button type="button" onClick={resetForm}>초기화</button>
    </form>
  );
}
```

`useForm` 훅은 폼 입력 상태와 변경 핸들러, 제출 핸들러, 초기화 함수를 제공하여 어떤 폼에서도 쉽게 재사용할 수 있도록 해줘요. `useCallback`을 사용하여 불필요한 함수 재생성을 막아 성능 최적화도 고려했답니다.

### 2️⃣ 브라우저 API 활용 훅 (useLocalStorage 예시)

브라우저의 `localStorage`와 같은 API를 직접 다루는 로직도 커스텀 훅으로 만들면 재사용성이 높아져요. `useLocalStorage` 훅은 특정 키에 해당하는 값을 `localStorage`에 저장하고 가져오는 기능을 추상화할 수 있어요.

```tsx showLineNumbers
import { useState, useEffect } from 'react';

function useLocalStorage<T>(key: string, initialValue: T): [T, (value: T) => void] {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error('Error reading localStorage key "' + key + '":', error);
      return initialValue;
    }
  });

  const setValue = (value: T) => {
    try {
      setStoredValue(value);
      window.localStorage.setItem(key, JSON.stringify(value));
    } catch (error) {
      console.error('Error setting localStorage key "' + key + '":', error);
    }
  };

  // 외부에서 localStorage가 변경되었을 때 상태를 동기화하는 효과
  useEffect(() => {
    const handleStorageChange = (event: StorageEvent) => {
      if (event.key === key && event.newValue !== null) {
        try {
          setStoredValue(JSON.parse(event.newValue));
        } catch (error) {
          console.error('Error parsing storage event value for key "' + key + '":', error);
        }
      } else if (event.key === key && event.newValue === null) {
        // 값이 삭제되었을 때
        setStoredValue(initialValue);
      }
    };

    window.addEventListener('storage', handleStorageChange);
    return () => {
      window.removeEventListener('storage', handleStorageChange);
    };
  }, [key, initialValue]);

  return [storedValue, setValue];
}

// 사용 예시
function ThemeSwitcher() {
  const [theme, setTheme] = useLocalStorage<'light' | 'dark'>('app-theme', 'light');

  const toggleTheme = () => {
    setTheme(prevTheme => (prevTheme === 'light' ? 'dark' : 'light'));
  };

  useEffect(() => {
    document.body.className = theme;
  }, [theme]);

  return (
    <div>
      <p>현재 테마: {theme}</p>
      <button onClick={toggleTheme}>테마 전환</button>
      <style>{`
        body.light { background-color: #f0f0f0; color: #333; }
        body.dark { background-color: #333; color: #f0f0f0; }
      `}</style>
    </div>
  );
}
```

`useLocalStorage` 훅은 `localStorage`에 값을 읽고 쓰는 로직을 추상화하여, 컴포넌트에서는 단순히 상태를 관리하듯이 사용할 수 있도록 해줘요. 또한, `storage` 이벤트를 리스닝하여 다른 탭이나 창에서 `localStorage` 값이 변경되었을 때도 상태를 동기화하는 고급 기능까지 포함하고 있어요. 이는 실무에서 매우 유용하게 활용될 수 있는 패턴이에요.

## 🧪 커스텀 훅 테스트 전략

커스텀 훅은 순수 JavaScript 함수에 가깝지만, React 훅의 규칙을 따르기 때문에 일반 함수처럼 직접 테스트하기는 어려워요. `react-hooks-testing-library`와 같은 도구를 사용하면 React 환경에서 훅을 격리하여 효과적으로 테스트할 수 있답니다.

### 0️⃣ `react-hooks-testing-library` 활용

이 라이브러리는 훅을 테스트하기 위한 유틸리티를 제공하여, 실제 컴포넌트를 렌더링하는 것과 유사한 환경에서 훅의 상태와 반환 값을 검증할 수 있게 해줘요.

*   **`renderHook`**: 훅을 렌더링하고, 훅의 반환 값과 리렌더링 함수를 제공해요.
*   **`act`**: React의 업데이트를 래핑하여, 테스트가 React의 스케줄러와 동기화되도록 해줘요.

<LinkPreviewCard href="https://react-hooks-testing-library.com/" />

### 1️⃣ Mocking과 의존성 주입

훅이 외부 API 호출이나 브라우저 API에 의존하는 경우, 테스트 시에는 Mocking을 사용하여 실제 환경에 의존하지 않도록 해야 해요. 예를 들어, `fetch` 함수를 Mocking하거나, 훅에 의존성을 인자로 주입하는 방식으로 테스트의 독립성을 확보할 수 있어요.

### 2️⃣ 예시 코드 (`useCounter` 훅과 테스트 코드)

간단한 카운터 훅을 만들고, 이를 테스트하는 예시를 통해 `react-hooks-testing-library`의 사용법을 알아볼게요.

**`useCounter.ts`**
```ts showLineNumbers
import { useState, useCallback } from 'react';

interface UseCounterResult {
  count: number;
  increment: () => void;
  decrement: () => void;
  reset: () => void;
}

export function useCounter(initialValue: number = 0): UseCounterResult {
  const [count, setCount] = useState(initialValue);

  const increment = useCallback(() => setCount(prevCount => prevCount + 1), []);
  const decrement = useCallback(() => setCount(prevCount => prevCount - 1), []);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);

  return { count, increment, decrement, reset };
}
```

**`useCounter.test.ts` (Vitest 환경 예시)**
```tsx showLineNumbers {5, 11-13, 16, 20-21, 25-26}
import { renderHook, act } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import { useCounter } from './useCounter';

describe('useCounter', () => {
  it('초기값으로 0을 반환해야 해요', () => {
    const { result } = renderHook(() => useCounter());
    expect(result.current.count).toBe(0);
  });

  it('초기값을 설정할 수 있어야 해요', () => {
    const { result } = renderHook(() => useCounter(10));
    expect(result.current.count).toBe(10);
  });

  it('increment 함수를 호출하면 카운트가 증가해야 해요', () => {
    const { result } = renderHook(() => useCounter(0));
    act(() => {
      result.current.increment();
    });
    expect(result.current.count).toBe(1);
  });

  it('decrement 함수를 호출하면 카운트가 감소해야 해요', () => {
    const { result } = renderHook(() => useCounter(5));
    act(() => {
      result.current.decrement();
    });
    expect(result.current.count).toBe(4);
  });

  it('reset 함수를 호출하면 초기값으로 돌아가야 해요', () => {
    const { result } = renderHook(() => useCounter(10));
    act(() => {
      result.current.increment(); // 11
      result.current.increment(); // 12
      result.current.reset();
    });
    expect(result.current.count).toBe(10);
  });

  it('새로운 초기값으로 리셋되어야 해요', () => {
    const { result, rerender } = renderHook(({ initialValue }) => useCounter(initialValue), {
      initialProps: { initialValue: 0 },
    });

    act(() => {
      result.current.increment(); // count: 1
    });
    expect(result.current.count).toBe(1);

    rerender({ initialValue: 100 }); // initialValue 변경

    act(() => {
      result.current.reset(); // 새로운 initialValue로 리셋
    });
    expect(result.current.count).toBe(100);
  });
});
```

테스트 코드에서 `renderHook`을 사용하여 `useCounter` 훅을 렌더링하고, `result.current`를 통해 훅의 반환 값에 접근하는 것을 볼 수 있어요. `act` 함수는 훅 내부에서 발생하는 상태 업데이트가 React의 렌더링 사이클에 맞춰 처리되도록 보장해 준답니다. 이렇게 하면 훅의 동작을 안정적으로 검증할 수 있어요.

## 🎯 커스텀 훅 사용 시 주의사항과 팁

커스텀 훅은 강력하지만, 잘못 사용하면 오히려 코드 베이스를 복잡하게 만들거나 성능 문제를 야기할 수 있어요. 몇 가지 주의사항과 팁을 알려드릴게요.

### 0️⃣ 과도한 추상화 피하기

모든 로직을 훅으로 만들 필요는 없어요. 로직이 너무 단순해서 재사용성이 없거나, 오직 하나의 컴포넌트에서만 사용되는 경우라면 굳이 커스텀 훅으로 분리하지 않는 것이 더 나을 수 있어요. 과도한 추상화는 오히려 코드의 복잡성을 높이고 이해하기 어렵게 만들 수 있답니다. 항상 '이 훅이 정말 필요한가?'를 자문해 보세요.

### 1️⃣ 훅의 이름 컨벤션

커스텀 훅은 항상 `use`로 시작하는 이름을 사용해야 해요 (예: `useToggle`, `useForm`). 이는 React의 훅 규칙이며, 개발자들이 해당 함수가 훅임을 쉽게 인지하고 훅 규칙(예: 컴포넌트 최상위 또는 다른 훅 내부에서만 호출)을 따르도록 유도하는 중요한 컨벤션이에요.

### 2️⃣ 훅 내부에서 다른 훅 호출

커스텀 훅은 `useState`, `useEffect`와 같은 기본 훅뿐만 아니라, 다른 커스텀 훅을 호출할 수도 있어요. 이는 더 복잡한 로직을 여러 개의 작은 훅으로 쪼개어 관리할 때 유용해요. 예를 들어, `useForm` 훅 내부에서 `useValidation` 훅을 호출하는 식으로요. 이렇게 하면 훅 간의 관심사 분리를 더욱 효과적으로 할 수 있답니다.
