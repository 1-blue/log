React 19의 Actions와 useFormStatus, useOptimistic 훅을 사용해서 복잡한 폼 처리 로직을 선언적으로 만들고, 사용자에게 즉각적인 피드백을 제공하는 UI를 구현하는 방법을 알아봐요.

## 🤔 문제/배경
### 0️⃣ 왜 이 주제를 다루는가
웹 애플리케이션에서 사용자로부터 데이터를 입력받는 폼은 핵심적인 요소예요. 하지만 폼 제출 시 서버와 통신하고, 로딩 상태를 표시하며, 에러를 처리하고, 최종적으로 UI를 업데이트하는 과정은 생각보다 복잡한 경우가 많아요.<br />
특히 비동기 처리와 UI 상태 관리를 동시에 고려해야 해서 초중급 개발자분들이 많이 어려워하는 부분이기도 해요.

React 19는 이러한 폼 처리 경험을 획기적으로 개선할 수 있는 새로운 기능인 "Actions"와 관련 훅들을 소개했어요. 이 기능들을 활용하면 서버와 클라이언트 간의 데이터 흐름을 더 선언적이고 효율적으로 관리할 수 있어서 개발 생산성을 높이고 사용자 경험을 향상시킬 수 있어요.

### 1️⃣ 기존 방식의 한계
기존 React 애플리케이션에서 폼을 처리할 때는 다음과 같은 방식들을 주로 사용했어요.<br />

*   **수동적인 상태 관리**: 폼 제출 시 `useState` 훅을 이용해 `isLoading`, `isError`, `data` 등의 상태를 직접 관리해야 했어요. 예를 들어, 제출 버튼 클릭 시 `isLoading(true)`로 설정하고, 서버 응답 후 `isLoading(false)`로 되돌리는 로직을 매번 작성해야 했죠.<br />
*   **클라이언트-서버 로직 분리**: 폼 데이터 유효성 검사, 서버로의 데이터 전송, 서버 응답 처리 등 클라이언트와 서버에서 수행해야 할 로직이 명확히 분리되지 않고 복잡하게 얽히는 경우가 많았어요.<br />
*   **낙관적 UI 업데이트의 어려움**: 사용자가 폼을 제출했을 때, 서버 응답을 기다리지 않고 즉시 UI를 업데이트하여 사용자에게 빠른 피드백을 주는 낙관적(Optimistic) UI 업데이트는 구현하기 까다로운 패턴 중 하나였어요. 서버 응답이 실패했을 경우 다시 UI를 롤백하는 로직까지 직접 구현해야 했거든요.<br />
*   **데이터 재검증의 번거로움**: 폼 제출 후 데이터 목록을 업데이트해야 할 때, 수동으로 데이터를 다시 불러오거나 캐시를 무효화하는 등의 추가적인 작업이 필요했어요. 이는 불필요한 네트워크 요청을 유발하거나 데이터 불일치를 초래할 수도 있었죠.

이러한 한계점들은 코드의 복잡성을 증가시키고, 버그 발생 가능성을 높이며, 궁극적으로는 개발 속도를 저하시키는 요인이 되었어요.

## ⚙️ 해결 방법
### 0️⃣ 핵심 아이디어
React 19의 Actions는 폼 제출과 같은 사용자 상호작용을 서버에서 실행되는 비동기 함수(Server Action)와 연결하는 강력한 메커니즘을 제공해요.<br />
이를 통해 폼 제출 시의 로딩 상태, 데이터 변경, 에러 처리 등을 React가 자동으로 관리해 줄 수 있게 되었어요.<br />
핵심 아이디어는 다음과 같아요.

1.  **Server Action과의 통합**: 폼의 `action` 속성에 비동기 함수를 직접 할당하여, 폼 제출 시 해당 함수가 서버에서 실행되도록 해요.<br />
2.  **`useFormStatus` 훅**: 폼의 상태(제출 중인지, 제출이 완료되었는지 등)를 자식 컴포넌트에서 쉽게 접근할 수 있게 해줘요. 이 훅을 사용하면 로딩 스피너나 버튼 비활성화와 같은 UI 피드백을 선언적으로 구현할 수 있어요.<br />
3.  **`useOptimistic` 훅**: 서버 응답을 기다리지 않고 즉시 UI를 업데이트하는 낙관적 UI 패턴을 간단하게 구현할 수 있게 해줘요. 예를 들어, 게시물에 좋아요를 누르면 즉시 좋아요 수가 증가한 것처럼 보이고, 실제 서버 응답이 오면 최종 상태를 반영하는 식이죠.<br />
4.  **자동적인 재검증 및 UI 업데이트**: Server Action이 성공적으로 완료되면, React는 자동으로 관련된 데이터를 다시 불러오거나 캐시를 무효화하여 UI를 최신 상태로 유지해 줘요. 이는 `revalidatePath`나 `revalidateTag`와 같은 Next.js의 캐싱 전략과도 긴밀하게 연결될 수 있어요.

### 1️⃣ 적용 방법
Actions를 적용하는 방법은 크게 세 가지 단계로 나눌 수 있어요.<br />

1.  **Server Action 정의**: 폼 데이터 처리를 담당할 비동기 함수를 정의해요. 이 함수는 `"use server";` 지시어를 포함하여 서버에서 실행되도록 표시해야 해요.<br />
2.  **`form` 요소에 `action` 연결**: 정의한 Server Action을 `<form>` 요소의 `action` 속성에 직접 연결해요. 이렇게 하면 폼 제출 시 자동으로 해당 Server Action이 호출돼요.<br />
3.  **`useFormStatus`와 `useOptimistic` 활용**: 폼 제출 상태에 따라 UI를 업데이트하거나 낙관적 UI를 구현하기 위해 `useFormStatus`와 `useOptimistic` 훅을 사용해요. 이 훅들은 폼 컴포넌트의 자식 컴포넌트에서 사용할 수 있어요.<br />

이 방법들을 통해 복잡한 비동기 로직과 UI 상태 관리를 React에게 맡기고, 개발자는 핵심 비즈니스 로직에 더 집중할 수 있게 돼요.

## 🧪 예시
### 0️⃣ 코드/설정 예시
가상의 할 일 목록(Todo List) 애플리케이션에서 새로운 할 일을 추가하는 폼을 React 19 Actions와 함께 구현해 볼게요.

먼저, 서버 액션을 정의할 파일(`app/actions.ts` 또는 `lib/actions.ts` 등)을 생성해요.

```ts showLineNumbers
"use server"; // 이 파일의 모든 함수는 서버에서 실행됨을 명시해요.

interface Todo {
  id: string;
  text: string;
  completed: boolean;
}

// 가상의 데이터베이스 역할을 할 배열이에요.
// 실제 프로젝트에서는 DB 통신 로직이 들어가겠죠?
const todos: Todo[] = [
  { id: "1", text: "React 19 Actions 학습하기", completed: false },
  { id: "2", text: "블로그 포스팅 작성하기", completed: true },
];

export async function addTodo(formData: FormData) {
  const newTodoText = formData.get("todoText") as string; // 폼 데이터에서 "todoText" 필드를 가져와요.

  if (!newTodoText || newTodoText.trim() === "") {
    throw new Error("할 일 내용을 입력해주세요."); // 유효성 검사 실패 시 에러를 던져요.
  }

  // 가상의 네트워크 지연을 흉내내요.
  await new Promise((resolve) => setTimeout(resolve, 1000));

  const newTodo: Todo = {
    id: String(todos.length + 1), // 간단한 ID 생성 로직이에요.
    text: newTodoText,
    completed: false,
  };

  todos.push(newTodo); // 가상 데이터에 추가해요.

  console.log("새로운 할 일 추가됨:", newTodo);

  // 실제 Next.js 환경에서는 여기서 revalidatePath('/todos') 등을 호출하여 캐시를 갱신할 수 있어요.
  // 현재 예시에서는 간단히 console.log로 대체해요.

  return newTodo; // 추가된 할 일 정보를 반환해요.
}

export async function getTodos(): Promise<Todo[]> {
  // 가상의 네트워크 지연을 흉내내요.
  await new Promise((resolve) => setTimeout(resolve, 500));
  return todos;
}
```

이제 이 Server Action을 사용하는 클라이언트 컴포넌트를 만들어 볼게요.
`useFormStatus`와 `useOptimistic`은 클라이언트 컴포넌트에서만 사용할 수 있으니 `"use client";` 지시어를 포함해야 해요.

```tsx showLineNumbers
"use client";

import { experimental_useFormStatus as useFormStatus, experimental_useOptimistic as useOptimistic } from "react";
import { addTodo, getTodos } from "./actions"; // 위에서 정의한 Server Action을 가져와요.
import { useEffect, useState } from "react";

interface Todo {
  id: string;
  text: string;
  completed: boolean;
}

function SubmitButton() {
  const { pending } = useFormStatus(); // 폼 제출 상태를 가져와요.

  return (
    <button type="submit" disabled={pending}>
      {pending ? "추가 중..." : "할 일 추가"}
    </button>
  );
}

export default function TodoList() {
  const [initialTodos, setInitialTodos] = useState<Todo[]>([]);
  // useOptimistic 훅을 사용하여 낙관적 UI 업데이트를 구현해요.
  // 첫 번째 인자는 현재 상태, 두 번째 인자는 상태를 업데이트할 함수예요.
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    initialTodos,
    (currentTodos: Todo[], newTodoText: string) => [
      ...currentTodos,
      { id: "temp-" + Date.now(), text: newTodoText, completed: false }, // 즉시 추가된 것처럼 보이는 임시 할 일이에요.
    ]
  );

  useEffect(() => {
    // 컴포넌트 마운트 시 초기 할 일 목록을 불러와요.
    getTodos().then(setInitialTodos);
  }, []);

  const handleAddTodo = async (formData: FormData) => {
    const newTodoText = formData.get("todoText") as string;
    if (!newTodoText || newTodoText.trim() === "") {
      alert("할 일 내용을 입력해주세요!");
      return;
    }

    addOptimisticTodo(newTodoText); // 서버 응답 전에 낙관적으로 UI를 업데이트해요.

    try {
      await addTodo(formData); // Server Action을 호출해요.
      // Server Action이 성공하면, React는 자동으로 useOptimistic의 초기 상태(initialTodos)를 갱신할 기회를 줘요.
      // 여기서는 getTodos()를 다시 호출하여 실제 최신 데이터를 가져와야 해요.
      // 실제 Next.js에서는 revalidatePath 등으로 캐시를 갱신하면 이 과정이 더 자동화될 수 있어요.
      getTodos().then(setInitialTodos); // 실제 데이터로 초기 상태를 업데이트해요.
    } catch (error: any) {
      alert("할 일 추가에 실패했어요: " + error.message);
      getTodos().then(setInitialTodos); // 에러 발생 시 원래 상태로 롤백해요.
    }
  };

  return (
    <div style={{ padding: "20px", maxWidth: "500px", margin: "0 auto", border: "1px solid #eee", borderRadius: "8px" }}>
      <h1>나의 할 일 목록</h1>
      <form action={handleAddTodo} style={{ display: "flex", gap: "10px", marginBottom: "20px" }}>
        <input type="text" name="todoText" placeholder="새로운 할 일을 입력하세요" style={{ flexGrow: 1, padding: "8px", borderRadius: "4px", border: "1px solid #ccc" }} />
        <SubmitButton />
      </form>
      <ul style={{ listStyle: "none", padding: 0 }}>
        {optimisticTodos.map((todo) => (
          <li key={todo.id} style={{ padding: "10px 0", borderBottom: "1px dashed #eee", color: todo.id.startsWith("temp-") ? "gray" : "inherit" }}>
            {todo.id.startsWith("temp-") ? `(임시) ${todo.text}` : todo.text}
            {todo.completed && <span style={{ marginLeft: "10px", color: "green" }}>✅</span>}
          </li>
        ))}
      </ul>
      <Blockquote type="info">
        `useFormStatus`와 `useOptimistic`은 현재 `experimental_` 접두사가 붙어 있어요. React 19 정식 릴리즈 시 변경될 수 있으니 공식 문서를 꼭 확인해 주세요.
      </Blockquote>
    </div>
  );
}
```

### 1️⃣ 적용 결과
위 코드를 실행하면 다음과 같은 사용자 경험을 제공할 수 있어요.<br />

1.  **폼 제출 시 로딩 상태**: "할 일 추가" 버튼을 누르면 `useFormStatus` 훅 덕분에 버튼 텍스트가 "추가 중..."으로 바뀌고 비활성화돼요. 사용자는 앱이 작동 중임을 명확히 인지할 수 있어요.<br />
2.  **낙관적 UI 업데이트**: `useOptimistic` 훅 덕분에 폼을 제출하는 즉시, 서버 응답을 기다리지 않고 새로운 할 일이 목록에 회색 텍스트로 `(임시)`와 함께 나타나요. 이로 인해 사용자에게는 매우 빠른 피드백을 제공하는 것처럼 느껴져요.<br />
3.  **실제 데이터 반영**: 약 1초 후 서버 액션이 완료되면, `addOptimisticTodo`에서 추가했던 임시 할 일은 실제 서버에서 온 데이터로 교체되거나 기존 임시 항목이 제거되고 진짜 항목이 추가되는 방식으로 UI가 최종 업데이트돼요. 만약 서버 액션이 실패하면, 임시 항목은 사라지고 원래 목록으로 되돌아가요.

이러한 방식은 사용자가 지루하게 기다리지 않고 앱이 반응성 있게 느껴지도록 만들어 사용자 만족도를 크게 높일 수 있어요.

## 📝 정리
### 0️⃣ 핵심 요약
React 19의 Actions와 새로운 훅들은 웹 폼 처리 및 비동기 데이터 업데이트 방식을 혁신적으로 개선했어요.<br />

*   **Actions**: 폼의 `action` 속성을 통해 서버 액션(Server Action)을 직접 연결하여, 폼 제출 로직을 서버와 클라이언트 간에 더 선언적으로 통합할 수 있게 해줘요.<br />
*   **`useFormStatus`**: 폼의 제출 상태(pending 여부)를 쉽게 파악하고, 이를 이용해 로딩 스피너나 버튼 비활성화와 같은 UI 피드백을 직관적으로 구현할 수 있도록 도와줘요.<br />
*   **`useOptimistic`**: 서버 응답을 기다리지 않고 즉시 UI를 업데이트하는 낙관적 UI 패턴을 간단하게 구현할 수 있게 하여, 사용자에게 더 빠르고 부드러운 경험을 제공해요.

이 기능들은 기존의 복잡했던 비동기 상태 관리 로직을 간소화하고, 개발자가 더 중요한 비즈니스 로직에 집중할 수 있도록 만들어요.<br />

### 1️⃣ 다음 액션
이 포스팅에서 다룬 React 19의 Actions와 훅들은 아직 실험적인(experimental) 단계에 있지만, 웹 개발의 미래 방향을 제시하고 있어요.<br />

1.  **공식 문서 확인**: React 19 정식 릴리즈 시 변경될 수 있으니, 항상 최신 React 공식 문서를 참고하여 정확한 사용법과 안정성을 확인해 주세요.<br />
2.  **Next.js 통합**: Next.js 14 이상 버전에서 Server Components 및 Server Actions와 함께 사용하면 더욱 강력한 시너지를 낼 수 있어요. Next.js App Router 환경에서 이 기능들을 직접 적용해 보는 것을 추천해 드려요.<br />
3.  **다양한 상황에 적용**: 단순 폼 제출뿐만 아니라, 좋아요 버튼 클릭, 댓글 추가, 아이템 삭제 등 서버와의 비동기 통신이 필요한 다양한 UI 패턴에 `useOptimistic`을 적용하여 사용자 경험을 개선하는 방법을 고민해 보세요.

새로운 React 19 기능들을 통해 더욱 효율적이고 사용자 친화적인 웹 애플리케이션을 만들어나가시길 바라요!
