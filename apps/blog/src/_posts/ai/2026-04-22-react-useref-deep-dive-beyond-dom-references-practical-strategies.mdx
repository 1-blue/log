## 🤔 useRef, 왜 알아야 할까요?

React에서 상태를 관리하는 가장 기본적인 훅은 `useState`예요.<br />
하지만 `useState`는 값이 변경될 때마다 컴포넌트를 리렌더링시키죠.<br />
모든 값이 리렌더링을 유발해야 하는 것은 아니에요.<br />
예를 들어, 타이머 ID, 스크롤 위치, 외부 라이브러리 인스턴스 등은 컴포넌트의 렌더링 결과에 직접적인 영향을 주지 않으면서도 컴포넌트 생명주기 동안 유지되어야 하는 값들이 많아요.

이럴 때 `useRef`가 빛을 발해요.<br />
`useRef`는 값이 변경되어도 컴포넌트를 리렌더링시키지 않으면서, 컴포넌트의 전 생애주기 동안 특정 값을 유지할 수 있게 해줘요.<br />
이는 특히 성능 최적화와 복잡한 로직 구현에 필수적인 요소가 된답니다.

## ⚙️ useRef의 기본 원리 이해하기

`useRef` 훅은 `current`라는 속성을 가진 변경 가능한(mutable) 객체를 반환해요.<br />
이 `current` 속성에 원하는 어떤 값이든 저장할 수 있어요.<br />
그리고 이 `current` 속성의 값이 변경되어도 React는 컴포넌트를 리렌더링하지 않아요.

### 0️⃣ useState vs useRef: 핵심 차이점 💡

`useState`와 `useRef`는 모두 컴포넌트 내에서 값을 "기억"하게 해준다는 공통점이 있지만, 중요한 차이점이 있어요.

<Blockquote type="info">
**useState:**<br />
- 값이 변경되면 컴포넌트가 리렌더링돼요.<br />
- 불변성(immutability)을 지향하며, 새로운 상태를 설정할 때마다 새로운 값을 사용해야 해요.<br />
- 주로 UI에 반영되어야 하는 상태(예: 카운터 값, 입력 필드 값)를 관리할 때 사용해요.
</Blockquote>

<Blockquote type="info">
**useRef:**<br />
- `current` 속성의 값이 변경되어도 컴포넌트가 리렌더링되지 않아요.<br />
- 변경 가능(mutable)한 객체를 반환하며, `current` 속성 값을 직접 수정할 수 있어요.<br />
- 주로 리렌더링을 유발하지 않으면서 컴포넌트 생애주기 동안 유지되어야 하는 값(예: DOM 요소 참조, 타이머 ID)을 관리할 때 사용해요.
</Blockquote>

```tsx showLineNumbers {5,10,14,18-19}
import React, { useState, useRef } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // count 변경 시 리렌더링
  const refCount = useRef(0); // refCount.current 변경 시 리렌더링 없음

  const incrementState = () => {
    setCount(prev => prev + 1);
  };

  const incrementRef = () => {
    refCount.current += 1; // current 값을 직접 수정
    console.log('Ref Count:', refCount.current); // 콘솔에는 변경된 값이 보이지만, UI는 리렌더링되지 않아요。
  };

  return (
    <div>
      <p>State Count: {count}</p>
      <button onClick={incrementState}>Increment State</button>
      <p>Ref Count (UI에 반영 안됨): {refCount.current}</p> {/* 이 값은 리렌더링되지 않아요 */}
      <button onClick={incrementRef}>Increment Ref (콘솔 확인)</button>
      <br /><br />
      <button onClick={() => setCount(0)}>Reset State to force Ref update</button>
      <p>강제로 리렌더링 시 Ref Count: {refCount.current}</p>
    </div>
  );
}

export default Counter;
```
위 예시에서 `Increment Ref` 버튼을 눌러보시면, 콘솔에는 `refCount.current` 값이 증가하는 것을 볼 수 있지만, UI의 "Ref Count" 부분은 업데이트되지 않는 것을 확인하실 수 있을 거예요.<br />
이는 `useRef`가 리렌더링을 유발하지 않기 때문이에요.<br />
만약 `Reset State` 버튼을 눌러 컴포넌트가 강제로 리렌더링되면 그제서야 `refCount.current`의 최신 값이 UI에 반영된답니다.

## 🚀 useRef 실전 활용 전략

이제 `useRef`의 다양한 실전 활용 방법을 알아볼게요.

### 1️⃣ DOM 요소에 직접 접근하기 (가장 흔한 용도) 🎯

`useRef`의 가장 기본적인 사용법은 특정 DOM 요소에 직접 접근하는 거예요.<br />
React는 선언적인(declarative) 방식의 UI 구축을 권장하지만, 때로는 특정 DOM 요소에 직접 접근하여 포커스를 주거나, 미디어 재생을 제어하거나, 스크롤 위치를 조작해야 할 필요가 있어요.

```tsx showLineNumbers {6,11-12,18}
import React, { useRef } from 'react';

function FocusInput() {
  const inputRef = useRef<HTMLInputElement>(null); // input 요소에 대한 참조를 저장할 ref 생성

  const handleClick = () => {
    // current 속성을 통해 DOM 요소에 접근
    if (inputRef.current) {
      inputRef.current.focus(); // input 요소에 포커스 부여
      inputRef.current.value = 'Focused!'; // 값도 직접 변경할 수 있어요
    }
  };

  return (
    <div>
      <input type="text" ref={inputRef} placeholder="여기에 포커스가 올 거예요" />
      <button onClick={handleClick}>Input에 포커스 주기</button>
    </div>
  );
}

export default FocusInput;
```
위 코드에서 `inputRef`를 `input` 요소의 `ref` 속성에 할당함으로써, `inputRef.current`를 통해 해당 DOM 요소에 접근할 수 있게 돼요.<br />
타입스크립트를 사용할 때는 `HTMLInputElement`와 같은 DOM 요소의 타입을 제네릭으로 지정하고, 초기값을 `null`로 설정하는 것이 좋아요.<br />
컴포넌트가 마운트되기 전에는 `current` 값이 `null`일 수 있기 때문에, 항상 `if (inputRef.current)`와 같이 null 체크를 해주는 습관을 들이는 것이 중요해요.

### 2️⃣ Mutable 값 저장: 리렌더링 없이 값 유지하기 💾

앞서 설명했듯이, `useRef`는 리렌더링 없이 변경 가능한 값을 저장할 수 있어요.<br />
이는 특히 타이머 ID, 이전 Props 값 저장, 스크롤 위치 추적 등에서 유용하게 사용돼요.

#### 2.1. 타이머 ID 관리 ⏰

`setInterval`이나 `setTimeout`과 같은 타이머 함수를 사용할 때, 컴포넌트가 언마운트될 때 타이머를 정리(cleanup)해야 메모리 누수를 방지할 수 있어요.<br />
이때 타이머 ID를 `useRef`에 저장하면 편리해요.

```tsx showLineNumbers {6,13,15-18,22-24,29}
import React, { useState, useRef, useEffect } from 'react';

function Timer() {
  const [count, setCount] = useState(0);
  const intervalRef = useRef<NodeJS.Timeout | null>(null); // 타이머 ID를 저장할 ref

  useEffect(() => {
    // 컴포넌트 마운트 시 타이머 시작
    intervalRef.current = setInterval(() => {
      setCount(prevCount => prevCount + 1);
    }, 1000);

    // 컴포넌트 언마운트 시 타이머 정리 (cleanup)
    return () => {
      if (intervalRef.current) {
        clearInterval(intervalRef.current);
        intervalRef.current = null; // 정리 후 null로 초기화
      }
    };
  }, []); // 빈 배열: 마운트 시 한 번만 실행

  const stopTimer = () => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
      console.log('타이머가 중지되었어요!');
    }
  };

  return (
    <div>
      <p>Current Count: {count}</p>
      <button onClick={stopTimer}>타이머 중지</button>
    </div>
  );
}

export default Timer;
```
`intervalRef.current`에 `setInterval`이 반환하는 ID를 저장하고, `useEffect`의 클린업 함수에서 이 ID를 사용하여 `clearInterval`을 호출해요.<br />
`intervalRef`의 값이 변경되어도 컴포넌트가 리렌더링되지 않으므로 성능에 영향을 주지 않으면서 타이머를 효과적으로 관리할 수 있어요.

#### 2.2. 이전 Props/State 값 저장 ↩️

`useRef`를 사용하면 컴포넌트가 리렌더링될 때 이전 Props나 State 값을 기억할 수 있어요.<br />
이는 특정 값이 변경되었을 때만 로직을 실행하거나, 이전 값과 현재 값을 비교해야 할 때 유용해요.

```tsx showLineNumbers {6,11-12,18}
import React, { useEffect, useRef } from 'react';

function PreviousValueDisplay({ value }: { value: number }) {
  const prevValueRef = useRef<number>(); // 이전 값을 저장할 ref

  useEffect(() => {
    // 현재 렌더링이 완료된 후에 이전 값을 업데이트
    prevValueRef.current = value;
  }); // 의존성 배열 없음: 모든 렌더링 후에 실행

  const previousValue = prevValueRef.current; // 이전 렌더링 시점의 값

  return (
    <div>
      <p>현재 값: {value}</p>
      <p>이전 값: {previousValue === undefined ? '아직 없음' : previousValue}</p>
      {value !== previousValue && previousValue !== undefined && (
        <p style={{ color: 'blue' }}>값이 {previousValue}에서 {value}로 변경되었어요!</p>
      )}
    </div>
  );
}

export default PreviousValueDisplay;
```
이 예제에서는 `useEffect` 훅을 사용하여 컴포넌트가 렌더링될 때마다 `prevValueRef.current`에 현재 `value`를 저장해요.<br />
`useEffect`는 모든 렌더링 이후에 실행되므로, `previousValue` 변수는 항상 "이전" 렌더링 시점의 `value`를 가리키게 된답니다.<br />
이를 통해 현재 값과 이전 값을 비교하는 로직을 쉽게 구현할 수 있어요.

<Blockquote type="tip">
**`useRef`와 `useEffect`의 상호작용 이해하기**<br />
`useRef`의 `current` 값은 컴포넌트의 렌더링 주기와 별개로 유지돼요.<br />
따라서 `useEffect` 내부에서 `ref.current` 값을 업데이트하면, 다음 렌더링 사이클에서 그 변경된 값을 사용할 수 있지만, 그 자체로는 리렌더링을 유발하지 않아요.<br />
이 점을 잘 활용하면 복잡한 상태 로직을 효율적으로 관리할 수 있어요.
</Blockquote>

### 3️⃣ 외부 라이브러리 통합: Canvas, D3 등 🎨

React 컴포넌트 내에서 Canvas, D3, Chart.js와 같은 DOM 기반의 외부 라이브러리를 사용할 때, 해당 라이브러리가 조작할 DOM 요소에 `useRef`를 통해 접근하는 것이 일반적인 패턴이에요.

```tsx showLineNumbers {6,10,13,16-20}
import React, { useRef, useEffect } from 'react';
import Chart from 'chart.js/auto'; // Chart.js v3+ import 방식

function ChartComponent() {
  const chartRef = useRef<HTMLCanvasElement>(null); // Canvas 요소 참조
  const chartInstance = useRef<Chart | null>(null); // Chart 인스턴스 참조

  useEffect(() => {
    if (chartRef.current) {
      // 기존 차트 인스턴스가 있으면 파괴
      if (chartInstance.current) {
        chartInstance.current.destroy();
      }

      // 새로운 차트 인스턴스 생성 및 저장
      chartInstance.current = new Chart(chartRef.current, {
        type: 'bar',
        data: {
          labels: ['Red', 'Blue', 'Yellow', 'Green', 'Purple', 'Orange'],
          datasets: [{
            label: '# of Votes',
            data: [12, 19, 3, 5, 2, 3],
            backgroundColor: [
              'rgba(255, 99, 132, 0.2)',
              'rgba(54, 162, 235, 0.2)',
              'rgba(255, 206, 86, 0.2)',
              'rgba(75, 192, 192, 0.2)',
              'rgba(153, 102, 255, 0.2)',
              'rgba(255, 159, 64, 0.2)'
            ],
            borderColor: [
              'rgba(255, 99, 132, 1)',
              'rgba(54, 162, 235, 1)',
              'rgba(255, 206, 86, 1)',
              'rgba(75, 192, 192, 1)',
              'rgba(153, 102, 255, 1)',
              'rgba(255, 159, 64, 1)'
            ],
            borderWidth: 1
          }]
        },
        options: {
          responsive: true,
          maintainAspectRatio: false,
        }
      });
    }

    // 컴포넌트 언마운트 시 차트 인스턴스 파괴
    return () => {
      if (chartInstance.current) {
        chartInstance.current.destroy();
      }
    };
  }, []); // 빈 배열: 마운트 시 한 번만 실행

  return (
    <div style={{ width: '600px', height: '400px' }}>
      <canvas ref={chartRef}></canvas>
    </div>
  );
}

export default ChartComponent;
```
이 예제에서는 `chartRef`를 `canvas` 요소에 연결하고, `useEffect` 내에서 `chartRef.current`를 사용하여 Chart.js 인스턴스를 생성해요.<br />
`chartInstance`도 `useRef`로 관리하여, 컴포넌트 언마운트 시 `destroy()` 메서드를 호출하여 메모리 누수를 방지하고 있어요.

### 4️⃣ 콜백 Ref 활용하기 (함수형 Ref) 🔄

`useRef`를 사용할 때, `ref` 속성에 직접 `useRef` 객체를 넘겨주는 대신 함수를 넘겨주는 '콜백 Ref' 방식을 사용할 수도 있어요.<br />
이 방식은 DOM 요소가 마운트되거나 언마운트될 때 추가적인 로직을 실행해야 할 때 유용해요.

```tsx showLineNumbers {6,8-14,19}
import React, { useRef, useCallback } from 'react';

function CallbackRefExample() {
  const measuredRef = useRef<HTMLDivElement | null>(null);

  // useCallback을 사용하여 콜백 함수가 리렌더링 시마다 재생성되는 것을 방지
  const setRef = useCallback((node: HTMLDivElement | null) => {
    // node는 ref가 연결될 DOM 요소 또는 null (언마운트 시)
    if (node) {
      measuredRef.current = node; // useRef에 값 저장
      console.log('DOM 요소가 마운트되었어요! 너비:', node.offsetWidth);
    } else {
      console.log('DOM 요소가 언마운트되었어요.');
    }
  }, []); // 의존성 배열이 비어있으므로 함수는 한 번만 생성

  const logWidth = () => {
    if (measuredRef.current) {
      console.log('현재 너비:', measuredRef.current.offsetWidth);
    }
  };

  return (
    <div>
      <div ref={setRef} style={{ width: '200px', height: '100px', border: '1px solid black' }}>
        콜백 Ref 예제 DIV
      </div>
      <button onClick={logWidth}>DIV 너비 출력</button>
    </div>
  );
}

export default CallbackRefExample;
```
`setRef` 함수는 DOM 요소가 마운트될 때 해당 요소(node)를 인자로 받아 실행되고, 언마운트될 때는 `null`을 인자로 받아 실행돼요.<br />
이 방식을 사용하면 `useEffect` 없이도 DOM 요소의 마운트/언마운트 시점을 감지하여 로직을 실행할 수 있어요.<br />
`useCallback`으로 `setRef` 함수를 메모이제이션하여 불필요한 함수 재생성을 막는 것이 일반적이에요.

## 📝 useRef 활용 팁 및 주의사항

### 0️⃣ `current` 속성 필수! ⚠️

`useRef`가 반환하는 객체 자체가 아닌, 그 객체의 `current` 속성을 통해 값에 접근하고 수정해야 해요.<br />
예를 들어, `myRef.current = value`는 올바르지만, `myRef = value`는 `myRef` 객체 자체를 덮어씌우는 것으로, React가 의도한 `useRef`의 동작 방식이 아니에요.

### 1️⃣ 리렌더링 유발하지 않음 🔄

`useRef.current` 값을 변경해도 컴포넌트가 리렌더링되지 않아요.<br />
만약 `useRef`에 저장된 값이 변경되었을 때 UI를 업데이트해야 한다면, `useState`를 사용하거나, `useRef`의 변경이 `useState`를 업데이트하는 트리거 역할을 하도록 설계해야 해요.

### 2️⃣ 초기값은 한 번만 설정 ☝️

`useRef(initialValue)`의 `initialValue`는 컴포넌트의 첫 번째 렌더링 시에만 사용돼요.<br />
이후 리렌더링 시에는 초기값이 무시되고, 이전에 저장된 `current` 값이 유지된답니다.

```tsx showLineNumbers {4}
import React, { useRef } from 'react';

function InitialValueExample({ initial }: { initial: number }) {
  const myRef = useRef(initial); // initial props는 첫 렌더링에만 사용

  // myRef.current는 리렌더링되어도 initial props의 변경에 영향을 받지 않아요.
  // 항상 첫 렌더링 시의 initial 값이나, 이후 수동으로 변경된 값을 유지해요.

  return (
    <div>
      <p>Props initial: {initial}</p>
      <p>Ref current: {myRef.current}</p>
      <button onClick={() => myRef.current++}>Ref 값 증가 (UI 변화 없음)</button>
    </div>
  );
}

export default InitialValueExample;
```
`InitialValueExample` 컴포넌트의 `initial` prop이 변경되어도 `myRef.current`는 첫 렌더링 시의 `initial` 값만 유지하는 것을 확인할 수 있어요.<br />
이는 `useRef`가 컴포넌트 인스턴스별로 고유한 값을 저장하기 때문이에요.

### 3️⃣ 클린업은 `useEffect`와 함께 🧹

타이머, 이벤트 리스너, 외부 라이브러리 인스턴스 등을 `useRef`로 관리할 때는 `useEffect`의 클린업 함수를 사용하여 컴포넌트가 언마운트될 때 리소스를 해제하는 것이 필수적이에요.<br />
이는 메모리 누수를 방지하고 애플리케이션의 안정성을 높이는 중요한 습관이에요.

## 📝 정리하며: useRef, 단순한 DOM 접근 그 이상!

오늘 우리는 React의 `useRef` 훅에 대해 깊이 있게 다뤄봤어요.<br />
`useRef`는 단순히 DOM 요소에 접근하는 용도를 넘어, 컴포넌트 생애주기 동안 변경 가능한 값을 유지하면서 리렌더링을 유발하지 않는다는 점에서 매우 강력한 도구예요.<br />
이를 통해 성능 최적화, 외부 라이브러리 통합, 복잡한 비동기 로직 관리 등 다양한 실전 상황에서 유용하게 활용할 수 있답니다.

핵심 내용을 다시 한번 요약해 드릴게요.

- **`useState` vs `useRef`**: `useState`는 상태 변화 시 리렌더링을 유발하고 불변성을 지향하지만, `useRef`는 `current` 값 변경 시 리렌더링이 없으며 변경 가능한(mutable) 값을 저장해요.
- **주요 활용 사례**: DOM 요소 직접 제어, 리렌더링 없이 값 유지 (타이머 ID, 이전 값), 외부 라이브러리 통합, 콜백 Ref를 통한 세밀한 제어.
- **주의사항**: 항상 `ref.current`로 접근하고, 리렌더링을 유발하지 않는다는 점을 인지하며, `useEffect` 클린업을 통해 리소스를 관리해야 해요.

`useRef`는 React 개발의 복잡성을 줄이고 성능을 향상시키는 데 큰 도움을 줄 수 있는 훅이에요.<br />
오늘 배운 내용을 바탕으로 여러분의 프로젝트에서 `useRef`를 더욱 효과적으로 활용해 보시길 바라요.<br />
