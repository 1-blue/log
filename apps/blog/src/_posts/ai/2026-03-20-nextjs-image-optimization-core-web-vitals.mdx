## 🖼️ 웹 이미지, 왜 중요할까요?

웹 페이지에서 이미지는 시각적인 정보를 전달하고 사용자 인터페이스를 풍부하게 만드는 핵심 요소예요. 하지만 잘못 관리된 이미지는 웹 성능 저하의 주범이 되기도 한답니다.

### 0️⃣ 이미지 최적화가 웹 성능에 미치는 영향

구글은 사용자 경험을 중요하게 생각하며, 이를 측정하는 핵심 지표로 **Core Web Vitals**를 제시하고 있어요. 이 지표들은 페이지 로딩 성능, 상호작용성, 시각적 안정성을 측정하는데요. 이미지는 이 지표들에 큰 영향을 미쳐요.

- **LCP (Largest Contentful Paint):** 페이지의 가장 큰 콘텐츠 요소가 렌더링되는 시간을 측정해요. 대부분의 경우, 페이지에서 가장 큰 콘텐츠는 이미지인 경우가 많아요. 최적화되지 않은 큰 이미지는 LCP를 지연시켜 사용자에게 답답함을 줄 수 있어요.
- **CLS (Cumulative Layout Shift):** 페이지 콘텐츠의 예기치 않은 레이아웃 이동량을 측정해요. 이미지의 `width`와 `height` 속성이 지정되지 않으면, 이미지가 로드된 후 갑자기 공간을 차지하며 주변 콘텐츠를 밀어내는 현상이 발생할 수 있어요. 이는 사용자 경험을 심각하게 해칠 수 있답니다.
- **FCP (First Contentful Paint):** 첫 번째 콘텐츠 요소가 렌더링되는 시간을 측정해요. 이미지 로딩이 느리면 FCP도 함께 지연될 수 있어요.

이처럼 이미지를 제대로 최적화하지 않으면 검색 엔진 순위에도 부정적인 영향을 미치고, 결국 사용자 이탈로 이어질 수 있다는 점을 꼭 기억해야 해요.

### 1️⃣ 기존 이미지 처리 방식의 한계

기존 웹 개발에서 이미지를 처리할 때 흔히 겪는 문제점들이 많아요. 일일이 수작업으로 최적화하기에는 너무 번거롭고 실수할 가능성도 높았어요.

- **수동 최적화의 어려움:** 각 기기 해상도에 맞는 다양한 크기의 이미지를 준비하고, 최신 이미지 포맷(WebP, AVIF)으로 변환하는 작업은 많은 시간과 노력을 필요로 해요.
- **레이지 로딩 구현의 번거로움:** 화면에 보이는 이미지부터 로드하고 보이지 않는 이미지는 나중에 로드하는 레이지 로딩(Lazy Loading)을 직접 구현하려면 Intersection Observer API를 사용해야 하는데, 이 또한 개발자에게 부담이 될 수 있어요.
- **CLS 문제:** `<img>` 태그에 `width`와 `height`를 명시하지 않으면, 이미지가 로드되기 전까지 브라우저는 이미지의 크기를 알 수 없어 레이아웃 쉬프트가 발생하기 쉬웠어요.
- **성능 측정 및 개선의 어려움:** 어떤 이미지가 LCP에 가장 큰 영향을 주는지 파악하고 개선하는 과정이 직관적이지 않았어요.

이러한 문제점들을 해결하기 위해 Next.js에서는 강력한 `Image` 컴포넌트를 제공하고 있어요.

## ✨ Next.js Image 컴포넌트의 마법

Next.js `Image` 컴포넌트(`next/image`)는 이러한 이미지 최적화의 어려움을 해결하기 위해 탄생했어요. 빌드 시점 또는 요청 시점에 이미지를 자동으로 최적화하여 성능과 개발 편의성을 동시에 잡을 수 있게 도와주죠.

### 0️⃣ 자동으로 얻는 성능 개선 효과

`Image` 컴포넌트를 사용하면 다음과 같은 이점을 자동으로 얻을 수 있어요.

- **반응형 이미지:** 기기 너비에 따라 최적화된 크기의 이미지를 제공해요.<br />
- **이미지 포맷 최적화:** WebP, AVIF와 같은 최신 이미지 포맷으로 자동 변환하여 파일 크기를 줄여줘요 (브라우저 지원 여부에 따라).<br />
- **레이지 로딩:** 뷰포트 밖에 있는 이미지는 자동으로 지연 로드하여 초기 로딩 시간을 단축시켜요.<br />
- **CLS 방지:** `width`와 `height` 속성을 필수로 요구하여 이미지 로드 전 레이아웃 공간을 확보, CLS를 방지해요.<br />
- **외부 이미지 최적화:** `next.config.js` 설정을 통해 외부 이미지도 최적화할 수 있어요.

이 모든 것이 `<img>` 태그 대신 `<Image />` 컴포넌트만 사용하면 자동으로 이루어진다는 점이 정말 매력적이에요.

### 1️⃣ 핵심 속성 파헤치기: `src`, `width`, `height`, `alt`

`Image` 컴포넌트를 사용하기 위한 가장 기본적인 속성들이에요.

- `src`: 이미지 파일의 경로를 지정해요. 로컬 이미지는 `import` 구문을 통해 사용하고, 외부 이미지는 URL 문자열로 지정해요.
- `width`, `height`: 이미지의 원본 너비와 높이를 픽셀 단위로 지정해요. 이 속성들은 CLS를 방지하기 위해 필수적이에요. `fill` 속성을 사용할 때는 이 두 속성을 생략할 수 있어요.
- `alt`: 이미지에 대한 설명을 제공하는 대체 텍스트예요. 웹 접근성(Accessibility)을 위해 반드시 포함해야 하는 필수 속성이랍니다.

```tsx showLineNumbers
import Image from "next/image";
import myLocalImage from "../public/my-local-image.jpg"; // 로컬 이미지 불러오기

export default function BasicImageUsage() {
  return (
    <div>
      {/* 로컬 이미지 사용 예시 */}
      <Image
        src={myLocalImage}
        alt="아름다운 자연 풍경"
        width={500} // 이미지의 원본 너비
        height={300} // 이미지의 원본 높이
      />

      {/* 외부 이미지 사용 예시 (next.config.js 설정 필요) */}
      <Image
        src="https://example.com/remote-image.jpg"
        alt="원격 서버의 이미지"
        width={600}
        height={400}
      />
    </div>
  );
}
```

### 2️⃣ 반응형 이미지를 위한 `sizes` 속성

`sizes` 속성은 브라우저에 뷰포트 크기에 따라 어떤 크기의 이미지를 로드해야 하는지 힌트를 제공해요. 이를 통해 브라우저는 가장 적절한 `srcset`을 선택하여 불필요하게 큰 이미지를 로드하는 것을 방지하고, LCP 개선에 도움을 줘요.

`sizes`는 미디어 쿼리 조건을 기반으로 이미지의 최종 렌더링 크기를 설명하는 문자열이에요. 예를 들어, `sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"`는 다음과 같은 의미를 가져요.

- 뷰포트 너비가 768px 이하일 때: 이미지 너비는 뷰포트 너비의 100%<br />
- 뷰포트 너비가 768px 초과 1200px 이하일 때: 이미지 너비는 뷰포트 너비의 50%<br />
- 뷰포트 너비가 1200px 초과일 때: 이미지 너비는 뷰포트 너비의 33%

`fill` 속성과 함께 사용하면 `width`, `height`를 지정하지 않고 부모 요소에 꽉 채우는 반응형 이미지를 만들 때 유용해요.

### 3️⃣ LCP 개선을 위한 `priority`와 `placeholder`

- `priority`: `true`로 설정하면 이미지가 LCP 요소로 간주되어, 페이지 로드 시 다른 리소스보다 먼저 로드되도록 최적화돼요. 뷰포트 내의 가장 중요한 이미지(예: 히어로 이미지)에만 `priority`를 `true`로 설정하는 것이 좋아요. 남용하면 오히려 성능에 부정적인 영향을 줄 수 있어요.
- `placeholder`: 이미지가 로드되기 전에 보여줄 임시 UI를 설정해요. `blur` 또는 `empty` 값을 가질 수 있어요. `blur`를 사용하면 `blurDataURL` 속성을 통해 흐릿한 이미지를 미리 보여주어 사용자 경험을 향상시킬 수 있어요.

```tsx showLineNumbers
import Image from "next/image";
import heroImage from "../public/hero-image.jpg"; // LCP 대상 이미지

export default function LCPImageExample() {
  return (
    <div>
      {/* LCP 개선을 위한 히어로 이미지 */}
      <Image
        src={heroImage}
        alt="메인 페이지의 중요한 이미지"
        width={1920}
        height={1080}
        priority={true} // 이 이미지를 최우선으로 로드해요
        placeholder="blur" // 이미지가 로드되기 전 흐릿한 효과를 보여줘요
      />

      {/* 일반 이미지 (레이지 로딩 기본 적용) */}
      <Image
        src="/secondary-image.jpg" // public 폴더 이미지
        alt="부가적인 이미지"
        width={800}
        height={600}
      />
    </div>
  );
}
```

### 4️⃣ 원격 이미지 및 커스텀 로더 설정

기본적으로 Next.js `Image` 컴포넌트는 로컬 이미지와 `public` 폴더 내의 이미지를 자동으로 최적화해요. 외부 URL의 이미지를 사용하려면 `next.config.js` 파일에 해당 도메인을 명시적으로 추가해야 해요.

```javascript showLineNumbers {2-7}
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    domains: ["example.com", "another-domain.com"], // 외부 이미지 도메인 등록
    // 또는 remotePatterns 사용 (권장)
    // remotePatterns: [
    //   {
    //     protocol: 'https',
    //     hostname: 'assets.example.com',
    //     port: '',
    //     pathname: '/my-assets/**',
    //   },
    // ],
  },
};

module.exports = nextConfig;
```

또한, 이미지 CDN(예: Cloudinary, Imgix)을 사용하거나 특정 로직으로 이미지를 로드해야 할 경우, `loader` 속성이나 `next.config.js`의 `images.loader` 설정을 통해 커스텀 로더를 정의할 수 있어요.

```tsx showLineNumbers
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    loader: "custom", // 커스텀 로더 사용을 명시해요
    loaderFile: "./my-image-loader.js", // 커스텀 로더 파일 경로
  },
};

module.exports = nextConfig;

// my-image-loader.js (예시)
export default function myImageLoader({ src, width, quality }) {
  const url = new URL(src);
  url.searchParams.set("w", width.toString());
  url.searchParams.set("q", (quality || 75).toString());
  return url.href;
}
```

## 🚀 실전 코드 예시로 배우는 Image 컴포넌트 활용

이제 실제 코드를 통해 `Image` 컴포넌트의 다양한 활용법을 살펴볼게요.

### 0️⃣ 기본적인 이미지 사용법

가장 기본적인 로컬 이미지 사용 예시예요. `width`와 `height`를 명시하여 CLS를 방지하고, 자동으로 레이지 로딩과 최적화를 적용해요.

```tsx showLineNumbers
// components/MyProductImage.tsx
import Image from "next/image";
import productThumbnail from "@/public/product-thumbnail.jpg"; // @/public는 alias 설정 가정

interface MyProductImageProps {
  altText: string;
}

export default function MyProductImage({ altText }: MyProductImageProps) {
  return (
    <div style={{ position: "relative", width: "200px", height: "150px" }}>
      <Image
        src={productThumbnail}
        alt={altText}
        width={200} // 원본 이미지의 너비를 지정해요
        height={150} // 원본 이미지의 높이를 지정해요
        className="rounded-lg shadow-md" // Tailwind CSS 클래스 예시
      />
    </div>
  );
}
```

### 1️⃣ 반응형 이미지 구현

`fill` 속성과 `sizes`를 함께 사용하여 부모 컨테이너에 맞춰 크기가 조절되는 반응형 이미지를 구현해 볼게요. 부모 요소는 `position: relative` 또는 `absolute`, `fixed`, `sticky` 속성을 가져야 해요.

```tsx showLineNumbers {12,13,14}
// components/ResponsiveHeroImage.tsx
import Image from "next/image";
import heroBanner from "@/public/hero-banner.jpg";

export default function ResponsiveHeroImage() {
  return (
    <div
      style={{
        width: "100%",
        height: "300px", // 부모 요소의 높이를 지정하여 이미지 공간을 확보해요
        position: "relative", // fill 속성을 위해 필수예요
        overflow: "hidden", // 이미지가 부모를 벗어나지 않도록 해요
      }}
    >
      <Image
        src={heroBanner}
        alt="반응형 웹사이트 배너"
        fill // 부모 요소에 꽉 채우도록 해요. width, height는 필요 없어요
        sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
        style={{ objectFit: "cover" }} // 이미지가 부모 요소를 채우도록 해요
        priority={true} // LCP에 영향을 줄 수 있으므로 우선순위를 높여줘요
        placeholder="blur"
      />
    </div>
  );
}
```

### 2️⃣ LCP 이미지 최적화

페이지 로드 시 가장 먼저 보이는 중요한 이미지에 `priority` 속성을 사용해 LCP를 개선하는 예시예요.

```tsx showLineNumbers {10}
// pages/index.tsx 또는 app/page.tsx
import Image from "next/image";
import mainHero from "@/public/main-hero.jpg";

export default function HomePage() {
  return (
    <main>
      <h1>블루의 기술 블로그에 오신 것을 환영해요!</h1>
      <Image
        src={mainHero}
        alt="블로그 메인 히어로 이미지"
        width={1400}
        height={700}
        priority={true} // 이 이미지가 LCP에 가장 중요하다고 브라우저에 알려줘요
        placeholder="blur"
      />
      <section>
        {/* ... 다른 콘텐츠들 ... */}
        <p>다양한 기술 이야기를 함께 나눠봐요.</p>
      </section>
    </main>
  );
}
```

### 3️⃣ 외부 이미지 도메인 설정

`next.config.js`에 외부 이미지 도메인을 추가하는 방법을 다시 한번 강조할게요. 이 설정 없이는 외부 이미지가 렌더링되지 않아요.

```diff-js showLineNumbers {4-7}
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
```

## 💡 Image 컴포넌트 활용 팁과 주의사항

`Image` 컴포넌트를 더욱 효과적으로 사용하기 위한 몇 가지 팁과 주의사항을 알려드릴게요.

### 0️⃣ 현대적인 이미지 포맷(WebP, AVIF) 활용

Next.js `Image` 컴포넌트는 브라우저가 지원하는 경우 WebP나 AVIF와 같은 최신 포맷으로 이미지를 자동으로 변환하여 제공해요. 이 포맷들은 JPEG나 PNG보다 훨씬 작은 파일 크기로 더 나은 품질을 제공하므로, 가능한 한 원본 이미지를 고품질로 제공하는 것이 좋아요. 컴포넌트가 알아서 최적의 포맷을 선택해 줄 거예요.

### 1️⃣ `alt` 속성, 절대 놓치지 마세요

`alt` 속성은 시각 장애가 있는 사용자를 위한 웹 접근성뿐만 아니라, 이미지가 로드되지 않았을 때 대체 텍스트를 제공하고 SEO에도 중요한 역할을 해요. 모든 `Image` 컴포넌트에 이미지의 내용을 명확하게 설명하는 `alt` 텍스트를 반드시 포함해 주세요.

### 2️⃣ `fill` 레이아웃 사용 시 부모 요소 스타일링

`fill` 속성을 사용할 때는 이미지가 부모 요소에 꽉 차도록 렌더링되므로, 부모 요소에 `position: relative` (또는 `absolute`, `fixed`, `sticky`)와 함께 `width`, `height`를 명시적으로 지정해 주는 것이 중요해요. 이렇게 해야 이미지가 올바른 크기와 위치에 렌더링되고, 레이아웃 쉬프트도 방지할 수 있어요.

```tsx showLineNumbers {3-7}
<div
  style={{
    position: "relative", // 필수
    width: "100%",
    height: "200px", // 부모 요소의 높이를 지정해야 이미지가 공간을 차지해요
  }}
>
  <Image
    src="/my-image.jpg"
    alt="꽉 채워진 이미지"
    fill
    style={{ objectFit: "cover" }}
  />
</div>
```

## 📝 마무리하며

오늘은 Next.js `Image` 컴포넌트를 활용한 이미지 최적화 전략에 대해 자세히 알아봤어요. 단순히 `<img>` 태그를 `<Image />`로 바꾸는 것만으로도 수많은 성능 개선 효과를 얻을 수 있다는 점이 정말 놀랍지 않나요?

### 0️⃣ 핵심 요약

- **Core Web Vitals 개선:** `Image` 컴포넌트는 LCP, CLS 등 핵심 웹 성능 지표를 자동으로 개선해 줘요.
- **자동 최적화:** 반응형 이미지, 최신 포맷 변환, 레이지 로딩 등 다양한 최적화가 기본으로 제공돼요.
- **필수 속성:** `src`, `width`, `height`, `alt`는 반드시 지정해야 해요.
- **반응형/LCP 최적화:** `sizes` 속성으로 반응형 이미지를, `priority` 속성으로 LCP 이미지를 최적화할 수 있어요.
- **외부 이미지:** `next.config.js`에 `domains` 또는 `remotePatterns` 설정을 잊지 마세요.

### 1️⃣ 다음 스텝

이제 여러분의 Next.js 프로젝트에 `Image` 컴포넌트를 적극적으로 적용해 볼 차례예요. 기존 `<img>` 태그들을 `<Image />`로 교체하고, `priority`, `sizes`, `placeholder` 등의 속성을 활용하여 웹 페이지의 성능을 측정해 보세요. 구글 Lighthouse나 PageSpeed Insights 같은 도구를 사용하면 개선 효과를 시각적으로 확인할 수 있을 거예요.

성능 최적화는 단순히 기술적인 작업을 넘어, 사용자에게 더 나은 경험을 제공하는 중요한 과정이에요. Next.js `Image` 컴포넌트와 함께 여러분의 웹사이트를 더욱 빠르고 매력적으로 만들어나가시길 응원해요!
