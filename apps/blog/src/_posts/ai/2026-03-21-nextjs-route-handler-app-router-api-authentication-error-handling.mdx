## 🚀 Next.js App Router의 Route Handler, 왜 중요한가요?

### 0️⃣ 기존 API Routes의 진화
Next.js를 사용해 보셨다면, `pages/api` 디렉토리에서 API Routes를 만들어 보셨을 거예요. 하지만 App Router 시대가 오면서 API를 만드는 방식도 진화했어요. 이제 `app` 디렉토리 안에서 Route Handler를 사용하게 되는데요.<br />
Route Handler는 웹 표준 Request, Response 객체를 기반으로 동작하며, Server Components와 마찬가지로 서버 환경에서만 실행돼요. 이를 통해 강력하고 유연한 API 엔드포인트를 구축할 수 있게 되었어요.

### 1️⃣ Route Handler의 핵심 장점
*   **웹 표준 기반:** Request, Response 객체를 사용하여 익숙하고 예측 가능한 방식으로 API를 다룰 수 있어요.<br />
*   **유연한 API 엔드포인트:** 특정 경로에 대한 GET, POST, PUT, DELETE 등 모든 HTTP 메서드를 지원해요.<br />
*   **Server Components와의 시너지:** 같은 서버 환경에서 동작하므로 Server Components와 데이터를 주고받기 용이해요.<br />
*   **캐싱 및 재검증:** Next.js의 강력한 캐싱 및 재검증 메커니즘과 통합되어 성능 최적화에 유리해요.

## ⚙️ Route Handler 기본 사용법과 HTTP 메서드 처리

### 0️⃣ 간단한 Route Handler 만들기
Route Handler는 `app` 디렉토리 내의 원하는 경로에 `route.ts` (또는 `.js`, `.tsx`) 파일을 생성하여 만들어요. 파일 이름이 `route.ts`여야 해요.<br />
각 HTTP 메서드에 해당하는 함수를 `export`하면 해당 메서드 요청을 처리할 수 있어요.

```tsx showLineNumbers
// app/api/products/route.ts
import { NextResponse } from 'next/server';

export async function GET(request: Request) {
  // URLSearchParams를 사용하여 쿼리 파라미터에 접근할 수 있어요.
  const { searchParams } = new URL(request.url);
  const id = searchParams.get('id');

  if (id) {
    // 특정 ID의 제품을 조회하는 로직
    return NextResponse.json({ message: `Product ID: ${id} 조회 성공했어요.` });
  }

  // 모든 제품 목록을 반환하는 로직
  const products = [
    { id: 1, name: 'Laptop', price: 1200 },
    { id: 2, name: 'Mouse', price: 25 },
  ];
  return NextResponse.json(products);
}

export async function POST(request: Request) {
  const body = await request.json(); // 요청 본문 파싱
  // 새로운 제품을 생성하는 로직
  console.log('새 제품 생성 요청:', body);
  return NextResponse.json({ message: '제품이 성공적으로 생성되었어요.', data: body }, { status: 201 });
}

export async function DELETE(request: Request) {
  const { searchParams } = new URL(request.url);
  const id = searchParams.get('id');

  if (!id) {
    return NextResponse.json({ message: '삭제할 제품의 ID가 필요해요.' }, { status: 400 });
  }

  // 제품 삭제 로직
  console.log(`제품 ID: ${id} 삭제 요청`);
  return NextResponse.json({ message: `제품 ID: ${id}가 성공적으로 삭제되었어요.` });
}
```

위 코드처럼 `GET`, `POST`, `DELETE` 함수를 정의하면 `/api/products` 경로로 들어오는 해당 HTTP 요청을 처리할 수 있어요.<br />
`NextResponse`는 Next.js에서 제공하는 확장된 Response 객체로, JSON 응답을 보내거나 리다이렉트하는 등의 편리한 기능을 제공해요.

### 1️⃣ 동적 세그먼트 활용
특정 ID를 가진 리소스에 접근할 때는 동적 세그먼트(`[id]`)를 사용할 수 있어요.

```tsx showLineNumbers
// app/api/posts/[id]/route.ts
import { NextResponse } from 'next/server';

interface Params {
  id: string;
}

export async function GET(request: Request, { params }: { params: Params }) {
  const id = params.id; // 동적 세그먼트 [id] 값에 접근

  // 특정 ID의 포스트를 조회하는 로직
  if (id === '1') {
    return NextResponse.json({ id: 1, title: '첫 번째 포스트', content: '안녕하세요, 블루입니다.' });
  } else if (id === '2') {
    return NextResponse.json({ id: 2, title: '두 번째 포스트', content: 'Route Handler는 편리해요.' });
  }

  return NextResponse.json({ message: '포스트를 찾을 수 없어요.' }, { status: 404 });
}
```

`route.ts` 파일의 두 번째 인자로 `{ params }` 객체를 받아 동적 세그먼트 값에 접근할 수 있어요.

## 🔒 Route Handler에 인증 및 권한 부여 추가하기

실제 애플리케이션에서는 API에 인증(Authentication)과 권한 부여(Authorization)가 필수적이에요. Route Handler는 미들웨어와 비슷하게 동작하는 방식으로 이를 처리할 수 있어요.

### 0️⃣ JWT 기반 인증 미들웨어 패턴
요청 헤더에서 JWT(JSON Web Token)를 추출하고 검증하는 로직을 추가하여 인증을 구현할 수 있어요.

```tsx showLineNumbers {11-28, 32}
// app/api/protected-data/route.ts
import { NextResponse } from 'next/server';
import jwt from 'jsonwebtoken'; // 실제 프로젝트에서는 더 강력한 라이브러리를 사용하세요.

const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key'; // 환경 변수 사용 권장

export async function GET(request: Request) {
  const authHeader = request.headers.get('Authorization');

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return NextResponse.json({ message: '인증 토큰이 필요해요.' }, { status: 401 });
  }

  const token = authHeader.split(' ')[1];

  try {
    const decoded = jwt.verify(token, JWT_SECRET); // 토큰 검증
    // 검증된 사용자 정보를 바탕으로 권한 부여 로직 추가 가능
    console.log('토큰 디코딩 성공:', decoded);

    // 인증 및 권한 부여가 완료되었다면 보호된 데이터를 반환해요.
    return NextResponse.json({
      message: '인증된 사용자에게만 제공되는 데이터입니다.',
      userId: (decoded as any).userId,
      data: ['Protected Item 1', 'Protected Item 2']
    });
  } catch (error) {
    console.error('JWT 검증 실패:', error);
    return NextResponse.json({ message: '유효하지 않은 토큰이에요.' }, { status: 403 });
  }
}
```
<Blockquote type="tip">
실제 프로덕션 환경에서는 `jsonwebtoken` 라이브러리 대신 `next-auth`와 같은 전문 인증 솔루션을 사용하거나, 더 강력한 JWT 라이브러리 및 보안 모범 사례를 따르는 것을 강력히 권장해요.
</Blockquote>

### 1️⃣ 커스텀 미들웨어 활용
Next.js 미들웨어(`middleware.ts`)를 사용하여 모든 Route Handler 요청 전에 인증 로직을 중앙 집중화할 수도 있어요.

```tsx showLineNumbers
// middleware.ts (루트 또는 app 디렉토리 바로 아래)
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  const token = request.cookies.get('accessToken')?.value; // 쿠키에서 토큰 가져오기

  // 특정 경로에 대해서만 인증을 요구할 수 있어요.
  if (request.nextUrl.pathname.startsWith('/api/protected')) {
    if (!token) {
      // 토큰이 없으면 로그인 페이지로 리다이렉트하거나 에러 응답
      return NextResponse.redirect(new URL('/login', request.url));
    }
    // 여기에서 JWT 검증 로직을 추가할 수도 있어요.
    // 유효하지 않은 토큰이면 에러 응답을 반환해요.
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/api/:path*'], // 모든 /api 경로에 미들웨어 적용
};
```
이 방식은 모든 `api` 경로에 대해 일괄적인 인증 및 권한 처리를 할 때 유용해요.

## 🚨 Route Handler 에러 처리 전략

API는 예상치 못한 상황에 대비한 견고한 에러 처리 로직이 필수적이에요. 사용자에게 적절한 피드백을 주고, 서버 로그를 통해 문제를 파악할 수 있도록 해야 해요.

### 0️⃣ 기본 에러 처리
`try...catch` 블록을 사용하여 예외를 잡고, `NextResponse.json`을 통해 적절한 HTTP 상태 코드와 메시지를 반환하는 것이 기본이에요.

```tsx showLineNumbers {12-16}
// app/api/items/route.ts
import { NextResponse } from 'next/server';

export async function GET(request: Request) {
  try {
    // 데이터베이스 조회 또는 외부 API 호출 로직
    const result = await fetch('https://api.example.com/data');
    if (!result.ok) {
      throw new Error('외부 API 호출에 실패했어요.');
    }
    const data = await result.json();
    return NextResponse.json(data);
  } catch (error) {
    console.error('데이터 조회 중 에러 발생:', error);
    // 에러 유형에 따라 다른 상태 코드를 반환할 수 있어요.
    return NextResponse.json(
      { message: '데이터를 가져오는 중 문제가 발생했어요.', error: (error as Error).message },
      { status: 500 }
    );
  }
}
```

### 1️⃣ 커스텀 에러 클래스 활용
좀 더 체계적인 에러 처리를 위해 커스텀 에러 클래스를 정의하고 활용할 수 있어요.

```tsx showLineNumbers
// lib/errors.ts
export class CustomError extends Error {
  statusCode: number;
  constructor(message: string, statusCode: number = 500) {
    super(message);
    this.name = 'CustomError';
    this.statusCode = statusCode;
  }
}

export class NotFoundError extends CustomError {
  constructor(message: string = '리소스를 찾을 수 없어요.') {
    super(message, 404);
    this.name = 'NotFoundError';
  }
}

export class UnauthorizedError extends CustomError {
  constructor(message: string = '인증되지 않은 사용자예요.') {
    super(message, 401);
    this.name = 'UnauthorizedError';
  }
}
```

```tsx showLineNumbers {18-20, 22-26}
// app/api/users/[id]/route.ts
import { NextResponse } from 'next/server';
import { CustomError, NotFoundError, UnauthorizedError } from '@/lib/errors'; // 정의한 커스텀 에러 임포트

interface Params {
  id: string;
}

export async function GET(request: Request, { params }: { params: Params }) {
  try {
    const userId = params.id;
    // 실제 데이터 조회 로직 (예시)
    if (userId === 'admin') {
      throw new UnauthorizedError('관리자 정보는 접근할 수 없어요.');
    }
    if (userId !== '123') {
      throw new NotFoundError(`사용자 ID ${userId}를 찾을 수 없어요.`);
    }

    return NextResponse.json({ id: userId, name: '블루', email: 'blue@example.com' });
  } catch (error) {
    if (error instanceof CustomError) {
      return NextResponse.json({ message: error.message }, { status: error.statusCode });
    }
    console.error('예상치 못한 에러 발생:', error);
    return NextResponse.json(
      { message: '서버 에러가 발생했어요.', error: (error as Error).message },
      { status: 500 }
    );
  }
}
```

이렇게 커스텀 에러 클래스를 사용하면 에러 유형에 따라 일관된 응답을 제공하고, 코드 가독성도 높일 수 있어요.

## ⚡️ 캐싱 전략과 Route Handler

Next.js App Router의 Route Handler는 강력한 캐싱 메커니즘과 긴밀하게 통합되어 있어요. 이를 잘 활용하면 애플리케이션의 성능을 크게 향상시킬 수 있어요.

### 0️⃣ `fetch` API의 캐싱 동작
Route Handler 내부에서 데이터를 가져올 때 사용하는 `fetch` API는 기본적으로 Next.js의 캐싱 메커니즘과 연동돼요.

```tsx showLineNumbers {8,11}
// app/api/cached-data/route.ts
import { NextResponse } from 'next/server';

export async function GET() {
  // 기본적으로 `force-cache` (정적 데이터) 또는 `no-store` (동적 데이터)처럼 동작해요.
  // revalidate 옵션을 통해 캐시 유효 기간을 설정할 수 있어요.
  const res = await fetch('https://api.example.com/posts', {
    next: { revalidate: 60 }, // 60초마다 데이터 재검증 (ISR과 유사)
  });

  if (!res.ok) {
    throw new Error('데이터를 가져오는 데 실패했어요.');
  }

  const data = await res.json();
  return NextResponse.json(data);
}
```
<Blockquote type="info">
`fetch`의 `next: { revalidate: N }` 옵션은 해당 데이터를 N초마다 재검증하도록 설정해요. 이는 ISR(Incremental Static Regeneration)과 유사하게 동작하여, 캐시된 데이터를 사용하면서도 주기적으로 최신 데이터를 반영할 수 있게 해줘요.
</Blockquote>

### 1️⃣ `revalidate` 옵션으로 캐시 제어
Route Handler 자체의 캐싱 동작을 제어하려면 `route.ts` 파일에 `export const revalidate = N` 또는 `export const dynamic = 'force-dynamic'` 등의 옵션을 사용할 수 있어요.

```tsx showLineNumbers {3-4}
// app/api/always-fresh-data/route.ts
import { NextResponse } from 'next/server';

export const dynamic = 'force-dynamic'; // 항상 동적으로 요청 처리 (캐시 사용 안 함)
// export const revalidate = 0; // 이와 동일한 효과를 낼 수 있어요. (기본값은 'force-cache' 또는 'auto')

export async function GET() {
  const timestamp = new Date().toISOString();
  return NextResponse.json({ message: '항상 최신 데이터에요!', timestamp });
}
```
*   `export const dynamic = 'force-dynamic'`: 이 Route Handler는 항상 동적으로 요청을 처리하며, 캐시를 사용하지 않아요.<br />
*   `export const revalidate = 0`: 이 또한 캐시를 사용하지 않고 매 요청마다 다시 데이터를 가져오도록 해요.<br />
*   `export const revalidate = false`: 캐시를 영원히 사용하지 않고, 빌드 시점에 생성된 데이터만 사용하도록 강제해요 (정적 렌더링).

<Blockquote type="warning">
`force-dynamic`이나 `revalidate = 0`을 남용하면 서버 부하가 증가하고 성능이 저하될 수 있어요. 데이터의 신선도 요구 사항에 맞춰 적절히 사용하는 것이 중요해요.
</Blockquote>

더 자세한 캐싱 전략에 대해서는 Next.js 공식 문서를 참고하시면 좋아요.

## 📝 정리하며: Route Handler 마스터하기

### 0️⃣ 핵심 요약
Next.js App Router의 Route Handler는 웹 표준 기반의 유연하고 강력한 API 엔드포인트를 제공해요. `route.ts` 파일에 HTTP 메서드별 함수를 `export`하여 쉽게 API를 구축할 수 있어요.<br />
인증을 위해서는 요청 헤더의 JWT를 검증하거나 `middleware.ts`를 활용할 수 있고요, 에러 처리는 `try...catch`와 커스텀 에러 클래스를 통해 견고하게 만들 수 있어요.<br />
또한, `fetch` API의 `revalidate` 옵션이나 Route Handler의 `export const dynamic` 옵션을 통해 캐싱 전략을 섬세하게 제어하여 성능을 최적화할 수 있답니다.

### 1️⃣ 다음 액션
이제 Route Handler의 기본 개념과 실용적인 활용법을 익히셨으니, 실제 프로젝트에 적용해 보시는 것이 중요해요.<br />
*   **작은 API부터 직접 만들어 보세요:** 사용자 인증, 게시글 CRUD 등 간단한 API부터 시작해 보세요.<br />
*   **보안에 유의하세요:** JWT 시크릿 키는 환경 변수로 관리하고, 민감한 정보는 클라이언트에 노출되지 않도록 주의하세요.<br />
*   **테스트 코드를 작성하세요:** Vitest나 Testing Library를 사용하여 Route Handler의 동작을 검증하는 테스트 코드를 작성해 보세요.

Route Handler는 Next.js App Router의 핵심 기능 중 하나예요. 꾸준히 학습하고 적용하면서 여러분의 풀스택 개발 역량을 한 단계 더 성장시켜 보시길 바라요!
