## 🚀 사용자 경험, 왜 버벅일까요?

### 0️⃣ 현대 웹 애플리케이션의 도전 과제

요즘 웹 애플리케이션은 단순한 정보 제공을 넘어, 복잡한 로직과 풍부한 인터랙션을 제공하는 경향이 있어요.<br />
실시간 검색, 대용량 데이터 필터링, 복잡한 차트 렌더링 등 사용자 인터랙션이 많아질수록 UI 업데이트에 필요한 연산량도 함께 증가하게 돼요.<br />
이러한 무거운 연산이 메인 스레드를 너무 오랫동안 점유하게 되면, 사용자 입력(키보드 타이핑, 마우스 클릭 등)에 대한 반응이 늦어져 "버벅인다"고 느끼게 된답니다.

### 1️⃣ React의 렌더링과 블로킹 업데이트

React는 기본적으로 "동기적(synchronous)"으로 렌더링을 처리해요.<br />
어떤 상태가 업데이트되면, React는 해당 컴포넌트와 그 자식 컴포넌트들을 다시 렌더링하고 DOM을 업데이트하는 과정을 한 번에 쭉 진행해요.<br />
이 과정이 길어지면 브라우저는 다른 작업을 할 수 없게 되고, 특히 사용자 입력 처리와 같은 중요한 작업까지 지연될 수 있어요.<br />
결과적으로, 사용자는 UI가 멈춘 것처럼 보이거나 입력이 씹히는 듯한 불쾌한 경험을 하게 되는 거죠.

```tsx showLineNumbers
import React, { useState } from "react";

const HeavyComputationList = ({ searchTerm }: { searchTerm: string }) => {
  const items = Array.from({ length: 20000 }, (_, i) => `Item ${i + 1}`);

  const filteredItems = items.filter((item) =>
    item.toLowerCase().includes(searchTerm.toLowerCase()),
  );

  return (
    <div>
      {filteredItems.map((item) => (
        <div key={item}>{item}</div>
      ))}
    </div>
  );
};

const BlockingExample = () => {
  const [search, setSearch] = useState("");

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    // 이 업데이트가 동기적으로 처리되어, HeavyComputationList의 렌더링이 완료될 때까지
    // 다음 키 입력이 지연될 수 있어요.
    setSearch(e.target.value);
  };

  return (
    <div>
      <h1>블로킹 업데이트 예시</h1>
      <input
        type="text"
        value={search}
        onChange={handleChange}
        placeholder="검색어를 입력하세요"
      />
      <p>현재 검색어: {search}</p>
      <HeavyComputationList searchTerm={search} />
    </div>
  );
};

export default BlockingExample;
```

위 코드 예시에서 `BlockingExample` 컴포넌트는 `HeavyComputationList`라는 무거운 연산을 수행하는 자식 컴포넌트를 가지고 있어요.<br />
`input`에 텍스트를 입력할 때마다 `setSearch`가 호출되고, 이로 인해 `HeavyComputationList`가 필터링 및 렌더링을 다시 시작해요.<br />
만약 `HeavyComputationList`가 수만 개의 아이템을 필터링하고 렌더링해야 한다면, `input`에 타이핑할 때마다 UI가 순간적으로 멈추는 현상을 경험할 수 있답니다.

## ✨ `useTransition`으로 긴급하지 않은 업데이트 처리하기

### 0️⃣ `useTransition`이란 무엇인가요?

React 18에서 도입된 `useTransition` 훅은 UI 업데이트를 "긴급한(urgent)" 업데이트와 "긴급하지 않은(non-urgent)" 업데이트로 구분하여 처리할 수 있게 해줘요.<br />
사용자 입력에 즉각적으로 반응해야 하는 업데이트(예: 인풋 필드 값 변경)는 긴급하게 처리하고,<br />
데이터 필터링 결과 렌더링처럼 시간이 좀 걸려도 괜찮은 업데이트는 긴급하지 않게 처리해서 메인 스레드를 블로킹하지 않도록 돕는 것이죠.<br />

`useTransition`을 사용하면 긴급하지 않은 업데이트가 진행 중일 때, React는 더 긴급한 업데이트(예: 사용자 입력)가 발생하면 현재 진행 중이던 긴급하지 않은 업데이트를 "중단"하고 긴급한 업데이트를 먼저 처리해요.<br />
이후 긴급한 업데이트가 완료되면 긴급하지 않은 업데이트를 다시 시작하거나 처음부터 다시 시도한답니다.

### 1️⃣ `useTransition` 활용 패턴

`useTransition` 훅은 `[isPending, startTransition]` 배열을 반환해요.<br />

- `isPending`: 긴급하지 않은 업데이트가 현재 진행 중인지 여부를 나타내는 불리언 값이에요.<br />
- `startTransition`: 긴급하지 않은 업데이트를 래핑(wrap)하는 함수예요. 이 함수 안에 상태 변경 로직을 넣으면 해당 업데이트는 긴급하지 않은 것으로 분류돼요.

```tsx showLineNumbers
import React, { useState, useTransition } from "react";

const HeavyComputationList = ({ searchTerm }: { searchTerm: string }) => {
  const items = Array.from({ length: 20000 }, (_, i) => `Item ${i + 1}`);

  const filteredItems = items.filter((item) =>
    item.toLowerCase().includes(searchTerm.toLowerCase()),
  );

  return (
    <div>
      {filteredItems.map((item) => (
        <div key={item}>{item}</div>
      ))}
    </div>
  );
};

const UseTransitionExample = () => {
  const [search, setSearch] = useState("");
  const [displaySearch, setDisplaySearch] = useState("");
  const [isPending, startTransition] = useTransition(); // useTransition 훅 사용

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    // 1. 긴급한 업데이트: input의 실제 값은 즉시 반영
    setSearch(e.target.value);

    // 2. 긴급하지 않은 업데이트: 무거운 리스트 렌더링은 startTransition으로 래핑
    startTransition(() => {
      setDisplaySearch(e.target.value);
    });
  };

  return (
    <div>
      <h1>`useTransition` 예시</h1>
      <input
        type="text"
        value={search}
        onChange={handleChange}
        placeholder="검색어를 입력하세요"
      />
      <p>현재 검색어: {search}</p>
      {/* isPending 값에 따라 로딩 스피너 등을 표시할 수 있어요 */}
      {isPending && <p>로딩 중...</p>}
      <HeavyComputationList searchTerm={displaySearch} />
    </div>
  );
};

export default UseTransitionExample;
```

위 `UseTransitionExample` 코드에서는 `input`의 `search` 상태는 즉시 업데이트하여 사용자가 타이핑하는 내용을 바로 볼 수 있게 해요.<br />
하지만 `HeavyComputationList`에 전달되는 `displaySearch` 상태는 `startTransition`으로 래핑하여 업데이트를 지연시켰어요.<br />
이로 인해 `input`에 타이핑할 때 UI가 버벅이는 현상 없이 부드럽게 글자가 입력되고, 리스트는 백그라운드에서 비동기적으로 업데이트된답니다.

### 2️⃣ `isPending`으로 로딩 상태 표시하기

`useTransition`이 반환하는 `isPending` 값은 긴급하지 않은 업데이트가 진행 중일 때 `true`가 돼요.<br />
이를 활용하여 사용자에게 현재 어떤 작업이 진행 중임을 시각적으로 알려줄 수 있어요.<br />
예를 들어, 검색 결과가 업데이트되는 동안 로딩 스피너나 메시지를 보여줄 수 있죠.<br />
이는 사용자에게 "아, 지금 뭔가 처리 중이구나"라는 인식을 주어 기다림에 대한 불만을 줄여줄 수 있답니다.

```diff-tsx showLineNumbers {24, 30-32}
 import React, { useState, useTransition } from 'react';

 const HeavyComputationList = ({ searchTerm }: { searchTerm: string }) => {
   const items = Array.from({ length: 20000 }, (_, i) => `Item ${i + 1}`);

   const filteredItems = items.filter(item =>
     item.toLowerCase().includes(searchTerm.toLowerCase())
   );

   return (
     <div>
       {filteredItems.map(item => (
         <div key={item}>{item}</div>
       ))}
     </div>
   );
 };

 const UseTransitionExample = () => {
   const [search, setSearch] = useState('');
   const [displaySearch, setDisplaySearch] = useState('');
   const [isPending, startTransition] = useTransition();

   const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
     setSearch(e.target.value);

     startTransition(() => {
       setDisplaySearch(e.target.value);
     });
   };

   return (
     <div>
       <h1>`useTransition` 예시</h1>
       <input type="text" value={search} onChange={handleChange} placeholder="검색어를 입력하세요" />
       <p>현재 검색어: {search}</p>
       {isPending && <p style={{ color: 'blue' }}>데이터를 불러오는 중...</p>}
       <HeavyComputationList searchTerm={displaySearch} />
     </div>
   );
 };

 export default UseTransitionExample;
```

위 `diff` 코드에서 `isPending && <p style={{ color: 'blue' }}>데이터를 불러오는 중...</p>}` 부분이 추가된 것을 볼 수 있어요.<br />
`isPending`이 `true`일 때 "데이터를 불러오는 중..."이라는 메시지를 보여줌으로써, 사용자에게 현재 UI 업데이트가 백그라운드에서 진행 중임을 명확하게 알려줄 수 있어요.

## 💡 `useDeferredValue`로 값의 지연 처리하기

### 0️⃣ `useDeferredValue`란 무엇인가요?

`useDeferredValue` 훅은 어떤 값의 "지연된(deferred)" 버전을 제공해줘요.<br />
이 훅은 마치 디바운싱(debouncing)과 비슷하게 작동하지만, React의 동시성 렌더링 시스템과 긴밀하게 통합되어 있어요.<br />
`useDeferredValue`는 "이 값은 즉시 필요하지 않으니, 더 중요한 업데이트가 있다면 먼저 처리하고 나중에 여유가 될 때 업데이트해 줘"라고 React에게 알려주는 역할을 해요.<br />

이 훅은 주로 부모 컴포넌트로부터 전달받은 `props`나 상태 값의 연산이 무거울 때 유용하게 사용돼요.<br />
특히, 자식 컴포넌트가 해당 값을 사용하여 무거운 렌더링을 수행하는 경우에 빛을 발하죠.

### 1️⃣ `useDeferredValue` 활용 패턴

`useDeferredValue`는 인자로 특정 값을 받고, 그 값의 지연된 버전을 반환해요.<br />

```tsx showLineNumbers
import React, { useState, useDeferredValue } from "react";

const HeavyComputationList = ({ searchTerm }: { searchTerm: string }) => {
  const items = Array.from({ length: 20000 }, (_, i) => `Item ${i + 1}`);

  // 의도적인 지연을 추가하여 무거운 연산을 시뮬레이션
  const startTime = performance.now();
  while (performance.now() - startTime < 5) {
    /* Do nothing for 5ms */
  }

  const filteredItems = items.filter((item) =>
    item.toLowerCase().includes(searchTerm.toLowerCase()),
  );

  return (
    <div>
      {filteredItems.map((item) => (
        <div key={item}>{item}</div>
      ))}
    </div>
  );
};

const UseDeferredValueExample = () => {
  const [search, setSearch] = useState("");
  const deferredSearch = useDeferredValue(search); // search 값의 지연된 버전을 생성

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    // input의 값은 즉시 업데이트되어 사용자에게 즉각적인 피드백을 제공해요.
    setSearch(e.target.value);
  };

  return (
    <div>
      <h1>`useDeferredValue` 예시</h1>
      <input
        type="text"
        value={search}
        onChange={handleChange}
        placeholder="검색어를 입력하세요"
      />
      <p>현재 검색어: {search}</p>
      {/* deferredSearch는 search가 업데이트된 후 잠시 지연되어 HeavyComputationList에 전달돼요 */}
      {search !== deferredSearch && (
        <p style={{ color: "orange" }}>결과 업데이트 대기 중...</p>
      )}
      <HeavyComputationList searchTerm={deferredSearch} />
    </div>
  );
};

export default UseDeferredValueExample;
```

`UseDeferredValueExample`에서 `input`의 `search` 상태는 즉시 업데이트되지만, `HeavyComputationList`에 전달되는 `deferredSearch`는 `search` 값이 업데이트된 후 React가 여유가 있을 때 지연되어 업데이트돼요.<br />
이로 인해 사용자가 `input`에 타이핑할 때 UI가 끊기지 않고 부드럽게 작동하며, 무거운 리스트 렌더링은 백그라운드에서 처리될 수 있어요.

### 2️⃣ `useTransition`과의 차이점

`useTransition`과 `useDeferredValue`는 모두 "긴급하지 않은" 업데이트를 처리하여 UI 응답성을 개선한다는 공통점이 있지만, 사용 방식과 목적에서 약간의 차이가 있어요.<br />

<Blockquote type="info">
  **`useTransition`:**
  <br />
  - 특정 상태 업데이트 로직 자체를 "긴급하지 않은" 것으로 표시할 때 사용해요.
  <br />
  - 주로 상태를 직접 변경하는 함수(`setState` 등)를 래핑할 때 사용해요.
  <br />
  - `isPending` 값을 통해 업데이트 진행 여부를 명시적으로 알 수 있어요.
  <br />- **예시:** 검색어 입력 시 `input` 값은 즉시 업데이트하고, 검색 결과
  목록 렌더링은 `startTransition`으로 묶어 지연 처리할 때.
</Blockquote>

<Blockquote type="info">
  **`useDeferredValue`:**
  <br />
  - 어떤 "값" 자체를 지연된 버전으로 만들고 싶을 때 사용해요.
  <br />
  - 주로 부모 컴포넌트에서 내려받은 `props`나 현재 상태 값의 연산이 무거운 자식
  컴포넌트에 전달될 때 유용해요.
  <br />
  - `isPending`과 같은 명시적인 로딩 상태를 제공하지 않지만, `value !==
  deferredValue`와 같은 방식으로 상태 변화를 감지할 수 있어요.
  <br />- **예시:** `input`에 입력된 검색어 값 자체를 `useDeferredValue`로
  감싸서, 무거운 필터링 로직이 이 지연된 값을 사용하도록 할 때.
</Blockquote>

간단히 말해, `useTransition`은 "**어떤 액션(상태 변경)을 지연시킬까?**"에 초점을 맞추고, `useDeferredValue`는 "**어떤 값을 지연시킬까?**"에 초점을 맞춘다고 이해하시면 좋아요.

## 🛠️ 실전 예제로 이해하는 최적화

이제 실제 시나리오에서 두 훅을 어떻게 적용할 수 있는지 구체적인 예시를 통해 살펴볼게요.<br />
대용량 데이터를 검색 필터링하는 컴포넌트를 가정해볼게요.

### 0️⃣ 검색 필터링 컴포넌트 최적화

사용자가 검색 `input`에 타이핑할 때마다 수만 개의 아이템을 필터링하고 렌더링해야 한다고 상상해 보세요.<br />
이러한 상황에서 `input`의 응답성과 필터링 결과의 업데이트 간의 균형을 맞추는 것이 중요해요.<br />

```tsx showLineNumbers
// HeavyFilterList.tsx
import React from "react";

interface HeavyFilterListProps {
  items: string[];
  filterTerm: string;
}

// 이 컴포넌트는 filterTerm이 변경될 때마다 무거운 필터링 및 렌더링 작업을 수행한다고 가정합니다.
const HeavyFilterList = ({ items, filterTerm }: HeavyFilterListProps) => {
  // 의도적인 지연 추가 (실제 애플리케이션에서는 복잡한 계산이나 대량의 DOM 조작이 될 수 있어요)
  const startTime = performance.now();
  while (performance.now() - startTime < 10) {
    /* Simulate heavy work */
  }

  const filteredItems = React.useMemo(() => {
    if (!filterTerm) return items.slice(0, 100); // 검색어가 없으면 초기 100개만 보여줌
    return items
      .filter((item) => item.toLowerCase().includes(filterTerm.toLowerCase()))
      .slice(0, 1000); // 최대 1000개까지만 보여줌
  }, [items, filterTerm]);

  return (
    <div
      style={{
        border: "1px solid #eee",
        padding: "10px",
        marginTop: "10px",
        maxHeight: "300px",
        overflowY: "auto",
      }}
    >
      <h3>필터링된 결과 ({filteredItems.length}개)</h3>
      {filteredItems.length === 0 && <p>검색 결과가 없어요.</p>}
      <ul>
        {filteredItems.map((item, index) => (
          <li key={index}>{item}</li>
        ))}
      </ul>
    </div>
  );
};

export default HeavyFilterList;
```

위 `HeavyFilterList` 컴포넌트는 `filterTerm`이 변경될 때마다 `useMemo`를 사용하여 필터링 작업을 수행해요.<br />
`useMemo`를 사용했음에도 불구하고, 필터링 대상 `items`의 수가 매우 많거나 `filterTerm`이 자주 변경되면 여전히 UI 블로킹이 발생할 수 있답니다.

### 1️⃣ `useTransition` 적용 예시

이제 이 검색 필터링 컴포넌트에 `useTransition`을 적용하여 `input`의 응답성을 개선해볼게요.

```tsx showLineNumbers {24, 26-28, 36}
import React, { useState, useTransition } from "react";
import HeavyFilterList from "./HeavyFilterList"; // 위에서 정의한 컴포넌트

const allItems = Array.from(
  { length: 50000 },
  (_, i) => `Product ${i + 1} - A very detailed description for item ${i + 1}`,
);

const SearchWithTransition = () => {
  const [inputValue, setInputValue] = useState(""); // input에 즉시 반영될 값
  const [filterQuery, setFilterQuery] = useState(""); // HeavyFilterList에 전달될 값
  const [isSearching, startTransition] = useTransition(); // useTransition 훅

  const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    setInputValue(e.target.value); // input 값은 즉시 업데이트

    // 긴급하지 않은 업데이트로 필터링 쿼리 업데이트를 래핑
    startTransition(() => {
      setFilterQuery(e.target.value);
    });
  };

  return (
    <div>
      <h2>🔍 `useTransition`을 이용한 검색 필터링</h2>
      <input
        type="text"
        value={inputValue}
        onChange={handleInputChange}
        placeholder="검색어를 입력하세요..."
        style={{ width: "300px", padding: "8px", fontSize: "16px" }}
      />
      {isSearching && (
        <p
          style={{ color: "blue", display: "inline-block", marginLeft: "10px" }}
        >
          검색 중...
        </p>
      )}
      <p>현재 입력: **{inputValue}**</p>
      <HeavyFilterList items={allItems} filterTerm={filterQuery} />
    </div>
  );
};

export default SearchWithTransition;
```

`SearchWithTransition` 컴포넌트에서는 `inputValue`는 사용자의 타이핑에 맞춰 즉시 업데이트되어 `input` 필드에 반영돼요.<br />
반면, `HeavyFilterList`에 전달되는 `filterQuery`는 `startTransition`으로 감싸져 긴급하지 않은 업데이트로 처리돼요.<br />
따라서 사용자는 `input`에 타이핑하는 동안에도 UI가 끊김 없이 부드럽게 반응하는 것을 경험할 수 있고, `isSearching` 상태를 통해 검색 중임을 인지할 수 있게 된답니다.

### 2️⃣ `useDeferredValue` 적용 예시

이번에는 동일한 검색 필터링 시나리오에 `useDeferredValue`를 적용해볼게요.

```tsx showLineNumbers {19, 27}
import React, { useState, useDeferredValue } from "react";
import HeavyFilterList from "./HeavyFilterList"; // 위에서 정의한 컴포넌트

const allItems = Array.from(
  { length: 50000 },
  (_, i) => `Product ${i + 1} - A very detailed description for item ${i + 1}`,
);

const SearchWithDeferredValue = () => {
  const [inputValue, setInputValue] = useState(""); // input에 즉시 반영될 값

  // inputValue의 지연된 버전을 생성. HeavyFilterList는 이 값을 사용.
  const deferredInputValue = useDeferredValue(inputValue);

  const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    setInputValue(e.target.value);
  };

  return (
    <div>
      <h2>💡 `useDeferredValue`를 이용한 검색 필터링</h2>
      <input
        type="text"
        value={inputValue}
        onChange={handleInputChange}
        placeholder="검색어를 입력하세요..."
        style={{ width: "300px", padding: "8px", fontSize: "16px" }}
      />
      {inputValue !== deferredInputValue && (
        <p
          style={{
            color: "orange",
            display: "inline-block",
            marginLeft: "10px",
          }}
        >
          결과 업데이트 대기 중...
        </p>
      )}
      <p>현재 입력: **{inputValue}**</p>
      {/* HeavyFilterList는 지연된 값을 사용하여 렌더링돼요 */}
      <HeavyFilterList items={allItems} filterTerm={deferredInputValue} />
    </div>
  );
};

export default SearchWithDeferredValue;
```

`SearchWithDeferredValue` 컴포넌트에서는 `inputValue`는 `input`에 즉시 반영되지만, `HeavyFilterList`에 전달되는 `filterTerm`으로는 `useDeferredValue(inputValue)`를 통해 생성된 `deferredInputValue`를 사용해요.<br />
이 방식 역시 `useTransition`과 유사하게 `input`의 응답성을 확보하면서 무거운 필터링 작업을 백그라운드에서 처리할 수 있게 해줘요.<br />
`inputValue !== deferredInputValue` 조건을 통해 지연된 업데이트가 진행 중임을 사용자에게 알릴 수도 있답니다.

## 📝 핵심 요약 및 다음 단계

### 0️⃣ `useTransition`과 `useDeferredValue`의 역할

두 훅 모두 React 애플리케이션의 UI 응답성을 크게 향상시키는 데 기여해요.<br />
핵심은 긴급한 업데이트(사용자 입력)와 긴급하지 않은 업데이트(무거운 렌더링)를 분리하여 처리함으로써, 사용자가 앱이 "멈췄다"고 느끼지 않도록 하는 것이죠.<br />

- **`useTransition`**: 특정 상태 업데이트를 비동기적으로 처리하여 UI를 블로킹하지 않도록 해요. 주로 "액션"을 래핑할 때 사용하고, `isPending`으로 로딩 상태를 명확히 알 수 있어요.<br />
- **`useDeferredValue`**: 어떤 값 자체를 지연된 버전으로 제공하여, 해당 값을 사용하는 무거운 컴포넌트의 렌더링을 지연시켜요. 주로 `props`나 상태 값에 적용하여 자식 컴포넌트의 렌더링 최적화에 사용돼요.<br />

### 1️⃣ 언제 어떤 훅을 사용해야 할까요?

<Blockquote type="tip">
  **`useTransition`을 사용하는 경우:**
  <br />
  - 사용자 인터랙션(클릭, 폼 제출 등)으로 인해 발생하는 상태 업데이트가 무거운
  렌더링을 유발할 때.
  <br />
  - 특정 "액션"의 결과를 백그라운드에서 처리하면서, 사용자에게 즉각적인 피드백을
  제공해야 할 때.
  <br />- `isPending` 상태를 통해 명확한 로딩 UI를 보여주고 싶을 때.
</Blockquote>

<Blockquote type="tip">
  **`useDeferredValue`를 사용하는 경우:**
  <br />
  - 부모 컴포넌트에서 전달받은 `props`나 로컬 상태 값이 무거운 자식 컴포넌트의
  렌더링을 유발할 때.
  <br />
  - 주로 `input` 필드의 값과 같이 자주 변경되는 값이지만, 이 값을 사용하는
  무거운 연산은 지연되어도 괜찮을 때.
  <br />- `useTransition`처럼 명시적인 `startTransition` 호출 없이 값의 변화를
  React에 맡기고 싶을 때.
</Blockquote>

두 훅은 상호 보완적으로 사용될 수 있으며, 여러분의 애플리케이션의 특성과 성능 병목 지점에 따라 적절히 선택하여 적용하는 것이 중요해요.<br />
React의 동시성 기능들을 잘 이해하고 활용하면, 사용자에게 훨씬 더 부드럽고 쾌적한 경험을 제공할 수 있답니다.<br />
