> 이 글에서는 Next.js 15의 발전된 Data Cache와 Server Actions를 통해 App Router 환경에서 데이터 변경 후 UI를 효율적으로 업데이트하고 사용자 경험을 향상시키는 방법을 심도 있게 다뤄요. 특히 `revalidatePath`와 `revalidateTag`를 활용한 캐시 무효화 전략에 집중해요.

## 🤔 문제/배경

### 0️⃣ 왜 이 주제를 다루는가

Next.js App Router는 강력한 데이터 페칭 및 캐싱 기능을 제공하지만, 실무에서는 데이터가 변경된 후 클라이언트 UI를 최신 상태로 유지하는 것이 여전히 도전적인 과제예요.<br />특히 사용자 인터랙션(예: 게시글 작성, 상품 구매)으로 인해 서버 데이터가 변경될 때, 기존에 캐싱된 데이터를 어떻게 효율적으로 무효화하고 새로운 데이터를 가져와 UI를 업데이트할지가 중요한 문제로 다뤄져요.<br />Next.js 15에서는 이러한 문제를 더욱 효과적으로 해결하기 위한 캐싱 전략과 Server Actions의 통합이 강조되고 있어요. 우리는 이 새로운 기능을 제대로 이해하고 활용하여 더욱 반응적이고 효율적인 웹 애플리케이션을 구축해야 해요.

### 1️⃣ 기존 방식의 한계

Next.js App Router는 기본적으로 `fetch` API를 확장하여 요청을 자동으로 캐싱해요. 이는 정적인 데이터를 보여주는 페이지에서는 매우 효과적이지만, 데이터가 자주 변경되는 경우에는 문제가 될 수 있어요.<br />

<Blockquote type="warning">
  **기존 방식의 주요 한계점**<br />
  <ul>
    <li>`fetch` 캐시의 불확실성: `fetch`는 기본적으로 캐싱되지만, 언제 어떻게 캐시가 무효화되는지 정확히 알기 어렵다는 점이 있었어요.</li>
    <li>수동적인 UI 업데이트: 서버에서 데이터가 변경된 후, 클라이언트에서 `router.refresh()`를 호출하여 페이지를 새로고침하거나 데이터를 다시 가져와야 하는 경우가 많았어요. 이는 번거롭고 사용자 경험을 저해할 수 있어요.</li>
    <li>Client Components와 Server Components 간의 동기화 문제: Server Components에서 데이터를 가져와 렌더링한 후, Client Components에서 발생한 이벤트로 서버 데이터가 변경되면, Server Components의 데이터를 어떻게 업데이트할지 명확하지 않았어요.</li>
  </ul>
</Blockquote>

이러한 한계는 개발자가 캐싱 전략을 수동으로 관리해야 하는 부담을 주었으며, 때로는 오래된 데이터가 사용자에게 노출되는 원인이 되기도 했어요.

## ⚙️ 해결 방법

### 0️⃣ 핵심 아이디어

Next.js 15는 향상된 **Data Cache**와 **Server Actions**를 긴밀하게 통합하여 이러한 데이터 동기화 문제를 해결하는 강력한 메커니즘을 제공해요.<br />핵심 아이디어는 클라이언트 측에서 서버 데이터를 변경하는 액션(예: Form 제출)이 발생했을 때, 해당 Server Action 내에서 **서버의 캐시를 직접 무효화**하고, Next.js가 자동으로 관련 UI를 업데이트하도록 하는 거예요.<br />이를 통해 클라이언트-서버 간의 복잡한 데이터 동기화 로직을 개발자가 직접 구현할 필요 없이, 선언적이고 효율적으로 캐시를 관리할 수 있게 돼요.

### 1️⃣ Next.js 15의 Data Cache 심화

Next.js는 `fetch` API를 래핑하여 서버 측에서 데이터를 캐싱해요. 이 캐시는 서버 컴포넌트, `Route Handlers`, `generateStaticParams` 등 다양한 곳에서 활용돼요.<br />`fetch` 요청 시 `cache` 옵션과 `revalidate` 옵션을 통해 캐싱 동작을 세밀하게 제어할 수 있어요.

```tsx showLineNumbers {5-7, 10-12}
// app/blog/page.tsx
async function getPosts() {
  // 기본적으로 'force-cache' (무기한 캐싱)
  // 재검증은 revalidatePath 또는 revalidateTag로 수동으로 처리
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();
  return posts;
}

async function getRecentPosts() {
  // 10초마다 재검증되는 캐시
  const res = await fetch('https://api.example.com/recent-posts', { next: { revalidate: 10 } });
  const recentPosts = await res.json();
  return recentPosts;
}

async function getNoCachePosts() {
  // 캐싱되지 않고, 요청 시마다 새로운 데이터를 가져옴
  const res = await fetch('https://api.example.com/no-cache-posts', { cache: 'no-store' });
  const noCachePosts = await res.json();
  return noCachePosts;
}
```

<Blockquote type="info">
  **Data Cache 동작 원리 요약**<br />
  <ul>
    <li>**`cache: 'force-cache'` (기본값):** 서버에 캐시가 존재하면 항상 캐시된 데이터를 사용해요. 캐시가 없으면 서버에서 데이터를 가져와 캐시하고, 다음 요청부터는 캐시된 데이터를 사용해요. 이 캐시는 수동으로 무효화될 때까지 유지돼요.</li>
    <li>**`cache: 'no-store'`:** 요청 시마다 항상 새로운 데이터를 서버에서 가져와요. 캐싱을 완전히 비활성화할 때 사용해요.</li>
    <li>**`next: { revalidate: N }`:** `N`초마다 캐시를 재검증해요. 처음 요청 시 데이터를 캐시하고, `N`초 이후 다음 요청이 오면 백그라운드에서 데이터를 재검증하고 UI를 업데이트해요. ISR(Incremental Static Regeneration)과 유사하게 동작해요.</li>
  </ul>
</Blockquote>

Next.js 15에서는 이러한 `fetch` 캐싱 전략이 Server Actions와 더 유기적으로 연결되어 캐시 무효화가 더욱 강력해졌어요.

### 2️⃣ Server Actions를 활용한 캐시 무효화

Server Actions는 서버에서 실행되는 비동기 함수로, 클라이언트에서 직접 호출할 수 있어요. Next.js 15에서는 이 Server Actions 내에서 `revalidatePath` 또는 `revalidateTag` 함수를 사용하여 Data Cache를 명시적으로 무효화할 수 있게 되었어요.<br />

#### `revalidatePath(path: string, type?: 'page' | 'layout')`

특정 경로(Path)와 관련된 모든 캐시된 데이터를 무효화해요. 해당 경로의 Server Components가 다시 렌더링되어 새로운 데이터를 가져오도록 트리거해요.<br />`type` 옵션으로 `'page'` (해당 페이지의 데이터만 무효화) 또는 `'layout'` (해당 페이지와 그 상위 레이아웃까지 모두 무효화)을 지정할 수 있어요.

```ts showLineNumbers {8}
// actions/post.ts
'use server'; // Server Action임을 명시

import { revalidatePath } from 'next/cache';
import { redirect } from 'next/navigation';

export async function createPost(formData: FormData) {
  const title = formData.get('title');
  const content = formData.get('content');
  // ... 데이터베이스에 게시글 저장 로직 ...
  console.log('게시글 저장 완료:', { title, content });

  revalidatePath('/blog'); // '/blog' 경로의 캐시를 무효화해요.
  redirect('/blog'); // 게시글 목록 페이지로 리다이렉트해요.
}
```

#### `revalidateTag(tag: string)`

특정 `fetch` 요청에 부여된 태그(Tag)와 관련된 모든 캐시된 데이터를 무효화해요. 이는 `revalidatePath`보다 더 세밀한 제어가 필요할 때 유용해요. 예를 들어, 여러 페이지에서 동일한 종류의 데이터를 사용하고 있을 때, 특정 데이터 그룹만 무효화하고 싶을 때 사용해요.<br />`fetch` 요청 시 `next: { tags: ['my-tag'] }` 옵션을 추가하여 태그를 지정할 수 있어요.

```ts showLineNumbers {8}
// actions/product.ts
'use server';

import { revalidateTag } from 'next/cache';

export async function updateProductPrice(productId: string, newPrice: number) {
  // ... 데이터베이스에서 상품 가격 업데이트 로직 ...
  console.log(`상품 ${productId}의 가격을 ${newPrice}로 업데이트 완료했어요.`);

  revalidateTag('products'); // 'products' 태그가 있는 모든 fetch 요청의 캐시를 무효화해요.
}
```

<Blockquote type="tip">
  **언제 `revalidatePath`를 사용하고, 언제 `revalidateTag`를 사용할까요?**<br />
  <ul>
    <li>**`revalidatePath`:** 특정 페이지나 경로 전체의 데이터가 변경되어 재로딩이 필요할 때 사용해요. 예를 들어, 게시글을 새로 작성하면 게시글 목록 페이지 전체를 업데이트해야 할 때 적합해요.</li>
    <li>**`revalidateTag`:** 여러 페이지나 컴포넌트에서 동일한 종류의 데이터를 사용하고 있고, 그 데이터 그룹만 선택적으로 업데이트해야 할 때 사용해요. 예를 들어, "상품 목록" 데이터를 여러 곳에서 보여주고 있는데, 특정 상품의 가격이 변경되었을 때 상품 데이터 전체를 무효화할 때 유용해요. 더 세밀하고 유연한 캐시 제어가 가능해요.</li>
  </ul>
</BlockQuote>

## 🧪 예시

이제 실제 코드를 통해 Server Actions와 캐시 무효화 전략을 어떻게 활용하는지 살펴봐요.

### 0️⃣ Server Actions를 이용한 게시글 생성 예시

게시글 작성 폼을 만들고, 폼 제출 시 Server Action을 호출하여 게시글을 생성하고 캐시를 무효화하는 예시예요.<br />이 예시에서는 `revalidatePath`를 사용하여 게시글 목록 페이지를 자동으로 업데이트해요.

```tsx showLineNumbers
// app/blog/page.tsx
import Link from 'next/link';

async function getPosts() {
  const res = await fetch('https://api.example.com/posts', { next: { tags: ['posts'] } }); // 'posts' 태그 추가
  if (!res.ok) {
    throw new Error('Failed to fetch posts');
  }
  return res.json();
}

export default async function BlogPage() {
  const posts = await getPosts();

  return (
    <div style={{ maxWidth: '800px', margin: '0 auto', padding: '20px' }}>
      <h1>📝 블로그 게시글 목록</h1>
      <Link href="/blog/new" style={{ display: 'inline-block', margin: '20px 0', padding: '10px 15px', backgroundColor: '#0070f3', color: 'white', borderRadius: '5px', textDecoration: 'none' }}>
        새 게시글 작성
      </Link>
      <ul>
        {posts.map((post: any) => (
          <li key={post.id} style={{ borderBottom: '1px solid #eee', padding: '10px 0', listStyle: 'none' }}>
            <h2>{post.title}</h2>
            <p>{post.content.substring(0, 100)}...</p>
          </li>
        ))}
      </ul>
    </div>
  );
}

// app/blog/new/page.tsx
import { createPost } from '@/actions/post'; // Server Action 임포트

export default function NewPostPage() {
  return (
    <div style={{ maxWidth: '600px', margin: '0 auto', padding: '20px' }}>
      <h1>새 게시글 작성</h1>
      <form action={createPost} style={{ display: 'flex', flexDirection: 'column', gap: '15px' }}>
        <div>
          <label htmlFor="title" style={{ display: 'block', marginBottom: '5px' }}>제목:</label>
          <input type="text" id="title" name="title" required style={{ width: '100%', padding: '8px', border: '1px solid #ccc', borderRadius: '4px' }} />
        </div>
        <div>
          <label htmlFor="content" style={{ display: 'block', marginBottom: '5px' }}>내용:</label>
          <textarea id="content" name="content" rows={5} required style={{ width: '100%', padding: '8px', border: '1px solid #ccc', borderRadius: '4px' }}></textarea>
        </div>
        <button type="submit" style={{ padding: '10px 15px', backgroundColor: '#28a745', color: 'white', border: 'none', borderRadius: '5px', cursor: 'pointer' }}>
          게시글 등록
        </button>
      </form>
    </div>
  );
}

// actions/post.ts (server-only)
'use server';

import { revalidatePath } from 'next/cache';
import { redirect } from 'next/navigation';

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string;
  const content = formData.get('content') as string;

  // 실제 데이터베이스 저장 로직 (예시)
  await new Promise(resolve => setTimeout(resolve, 1000)); // DB 작업 시뮬레이션
  const newPost = { id: Date.now().toString(), title, content };
  console.log('새 게시글 저장 완료:', newPost);

  // 캐시 무효화 및 리다이렉트
  revalidatePath('/blog'); // '/blog' 경로의 캐시를 무효화하여 게시글 목록을 업데이트해요.
  redirect('/blog'); // 게시글 목록 페이지로 이동해요.
}
```
<Blockquote type="success">
  **적용 결과:**<br />
  `/blog/new` 페이지에서 게시글을 작성하고 '게시글 등록' 버튼을 누르면, `createPost` Server Action이 실행돼요.<br />
  게시글 저장 후 `revalidatePath('/blog')`가 호출되어 `/blog` 경로의 캐시가 무효화되고, `redirect('/blog')`로 인해 `/blog` 페이지로 이동할 때 Next.js는 새로운 데이터를 `getPosts` 함수를 통해 가져와 최신 게시글 목록을 보여줘요. 개발자가 명시적으로 `router.refresh()`를 호출할 필요가 없어지는 거죠.
</Blockquote>

### 1️⃣ `revalidateTag`를 이용한 상품 목록 업데이트 예시

상품의 가격을 업데이트하는 Server Action을 만들고, `revalidateTag`를 사용하여 특정 태그가 지정된 상품 목록의 캐시를 무효화하는 예시예요.<br />이 예시는 `revalidatePath`보다 더 세밀한 제어가 필요할 때 유용해요.

```tsx showLineNumbers {12, 34-36}
// app/products/page.tsx
import Link from 'next/link';

async function getProducts() {
  // 'products' 태그를 사용하여 이 fetch 요청을 식별해요.
  const res = await fetch('https://api.example.com/products', { next: { tags: ['products'] } });
  if (!res.ok) {
    throw new Error('Failed to fetch products');
  }
  return res.json();
}

export default async function ProductsPage() {
  const products = await getProducts();

  return (
    <div style={{ maxWidth: '800px', margin: '0 auto', padding: '20px' }}>
      <h1>🛍️ 상품 목록</h1>
      <ul>
        {products.map((product: any) => (
          <li key={product.id} style={{ borderBottom: '1px solid #eee', padding: '10px 0', listStyle: 'none', display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
            <div>
              <h2>{product.name}</h2>
              <p>가격: {product.price}원</p>
            </div>
            <Link href={`/products/${product.id}/edit`} style={{ padding: '8px 12px', backgroundColor: '#ffc107', color: 'white', borderRadius: '5px', textDecoration: 'none' }}>
              수정
            </Link>
          </li>
        ))}
      </ul>
    </div>
  );
}

// app/products/[id]/edit/page.tsx
import { updateProductPrice } from '@/actions/product';

export default async function EditProductPage({ params }: { params: { id: string } }) {
  const productId = params.id;
  // 실제 DB에서 상품 정보를 가져오는 로직 (예시)
  const product = { id: productId, name: `상품 ${productId}`, price: 10000 }; 

  return (
    <div style={{ maxWidth: '600px', margin: '0 auto', padding: '20px' }}>
      <h1>{product.name} 가격 수정</h1>
      <form action={updateProductPrice} style={{ display: 'flex', flexDirection: 'column', gap: '15px' }}>
        <input type="hidden" name="productId" value={productId} />
        <div>
          <label htmlFor="newPrice" style={{ display: 'block', marginBottom: '5px' }}>새로운 가격:</label>
          <input type="number" id="newPrice" name="newPrice" defaultValue={product.price} required style={{ width: '100%', padding: '8px', border: '1px solid #ccc', borderRadius: '4px' }} />
        </div>
        <button type="submit" style={{ padding: '10px 15px', backgroundColor: '#007bff', color: 'white', border: 'none', borderRadius: '5px', cursor: 'pointer' }}>
          가격 업데이트
        </button>
      </form>
    </div>
  );
}

// actions/product.ts (server-only)
'use server';

import { revalidateTag } from 'next/cache';
import { redirect } from 'next/navigation';

export async function updateProductPrice(formData: FormData) {
  const productId = formData.get('productId') as string;
  const newPrice = parseInt(formData.get('newPrice') as string, 10);

  // 실제 데이터베이스 업데이트 로직 (예시)
  await new Promise(resolve => setTimeout(resolve, 1000));
  console.log(`상품 ${productId}의 가격을 ${newPrice}로 업데이트 완료했어요.`);

  revalidateTag('products'); // 'products' 태그를 가진 모든 fetch 요청의 캐시를 무효화해요.
  redirect('/products'); // 상품 목록 페이지로 이동해요.
}
```
<Blockquote type="success">
  **적용 결과:**<br />
  `/products/[id]/edit` 페이지에서 상품 가격을 수정한 후 '가격 업데이트' 버튼을 누르면, `updateProductPrice` Server Action이 실행돼요.<br />
  가격 업데이트 후 `revalidateTag('products')`가 호출되어 'products' 태그가 지정된 `fetch` 요청의 캐시가 무효화돼요. `/products` 페이지로 리다이렉트될 때, Next.js는 `getProducts` 함수를 통해 새로운 상품 데이터를 가져와 최신화된 가격을 보여주게 돼요.
</Blockquote>

### 2️⃣ Optimistic UI 구현 (선택 사항)

사용자 경험을 더욱 향상시키기 위해, Server Actions와 함께 React의 `useOptimistic` 훅을 사용하여 Optimistic UI 패턴을 구현할 수 있어요.<br />Optimistic UI는 서버 응답을 기다리지 않고, 클라이언트에서 즉시 UI를 업데이트하여 사용자에게 즉각적인 피드백을 제공하는 기법이에요. 이후 서버에서 에러가 발생하면 UI를 이전 상태로 되돌릴 수 있어요.

```tsx showLineNumbers {1-2, 11-12, 17-21, 23-28}
// app/comments/page.tsx
'use client'; // useOptimistic은 클라이언트 훅이므로 'use client'가 필요해요.

import { useState, useRef, useOptimistic } from 'react';
import { addComment } from '@/actions/comment'; // 서버 액션

interface Comment {
  id: string;
  text: string;
}

export default function CommentsPage({ initialComments }: { initialComments: Comment[] }) {
  const [comments, setComments] = useState(initialComments);
  const formRef = useRef<HTMLFormElement>(null);

  // useOptimistic 훅을 사용하여 optimistic comments 상태를 관리해요.
  const [optimisticComments, addOptimisticComment] = useOptimistic(
    comments, // 초기 상태
    (currentComments, newCommentText: string) => [ // newCommentText는 addOptimisticComment의 인자
      ...currentComments,
      { id: 'optimistic-id-' + Date.now(), text: newCommentText, pending: true }, // 임시 ID와 pending 상태 추가
    ]
  );

  async function handleSubmit(formData: FormData) {
    const commentText = formData.get('comment') as string;
    formRef.current?.reset(); // 폼 초기화

    // Optimistic UI 업데이트: 서버 응답을 기다리지 않고 UI를 즉시 업데이트해요.
    addOptimisticComment(commentText); 

    try {
      // 서버 액션 호출
      const newComment = await addComment(formData);
      setComments((prev) => prev.map(c => c.id === 'optimistic-id-' + Date.now() ? newComment : c)); // 실제 ID로 교체
    } catch (error) {
      console.error('댓글 추가 실패:', error);
      // 에러 발생 시 Optimistic 업데이트를 되돌리거나 사용자에게 알림
      setComments(initialComments); // 예시: 초기 상태로 되돌림 (더 정교한 롤백 필요)
    }
  }

  return (
    <div style={{ maxWidth: '600px', margin: '0 auto', padding: '20px' }}>
      <h1>💬 댓글 목록</h1>
      <ul style={{ listStyle: 'none', padding: 0 }}>
        {optimisticComments.map((comment) => (
          <li key={comment.id} style={{ borderBottom: '1px solid #eee', padding: '10px 0', opacity: (comment as any).pending ? 0.6 : 1 }}>
            {comment.text} {(comment as any).pending && <span style={{ fontStyle: 'italic', color: '#888' }}>(전송 중...)</span>}
          </li>
        ))}
      </ul>
      <form ref={formRef} action={handleSubmit} style={{ display: 'flex', flexDirection: 'column', gap: '15px', marginTop: '30px' }}>
        <div>
          <label htmlFor="comment" style={{ display: 'block', marginBottom: '5px' }}>새 댓글:</label>
          <textarea id="comment" name="comment" rows={3} required style={{ width: '100%', padding: '8px', border: '1px solid #ccc', borderRadius: '4px' }} />
        </div>
        <button type="submit" style={{ padding: '10px 15px', backgroundColor: '#6f42c1', color: 'white', border: 'none', borderRadius: '5px', cursor: 'pointer' }}>
          댓글 추가
        </button>
      </form>
    </div>
  );
}

// actions/comment.ts (server-only)
'use server';

import { revalidatePath } from 'next/cache';

export async function addComment(formData: FormData) {
  const commentText = formData.get('comment') as string;

  // 실제 데이터베이스 저장 로직 (예시: 1.5초 지연)
  await new Promise(resolve => setTimeout(resolve, 1500));
  const newComment = { id: Math.random().toString(36).substring(2, 9), text: commentText };
  console.log('새 댓글 저장 완료:', newComment);

  revalidatePath('/comments'); // 댓글 목록 페이지 캐시 무효화
  return newComment; // 실제 저장된 댓글 객체를 반환해요.
}
```
<Blockquote type="tip">
  **적용 결과:**<br />
  댓글 입력 후 '댓글 추가' 버튼을 누르면, `useOptimistic` 훅 덕분에 즉시 "전송 중..." 상태의 댓글이 UI에 표시돼요. 사용자는 마치 댓글이 바로 추가된 것처럼 느끼게 돼요.<br />
  이후 서버 액션(`addComment`)이 성공적으로 완료되면, `setComments`를 통해 `pending` 상태가 제거되고 실제 서버에서 발급받은 ID로 교체되어 최종 UI가 확정돼요. 만약 서버 액션에서 에러가 발생했다면, `try-catch` 블록 내에서 `optimisticComments`를 이전 상태로 되돌릴 수 있어요.
</Blockquote>

## 📝 정리

### 0️⃣ 핵심 요약

Next.js 15의 Data Cache와 Server Actions는 App Router에서 데이터 변경 후 UI 업데이트를 처리하는 방식에 혁신을 가져왔어요.<br />

<Blockquote type="info">
  **핵심 정리**<br />
  <ul>
    <li>Next.js의 `fetch` API는 강력한 캐싱 기능을 제공하며, `cache` 및 `revalidate` 옵션으로 캐싱 전략을 세밀하게 제어할 수 있어요.</li>
    <li>Server Actions는 클라이언트에서 서버 데이터를 변경하는 액션을 수행할 때 유용하며, 이 액션 내에서 `revalidatePath`와 `revalidateTag`를 사용하여 서버 캐시를 명시적으로 무효화할 수 있어요.</li>
    <li>`revalidatePath`는 특정 경로의 캐시를 무효화하여 해당 경로의 Server Components가 다시 데이터를 가져오도록 만들어요.</li>
    <li>`revalidateTag`는 `fetch` 요청에 부여된 특정 태그 기반으로 캐시를 무효화하여 더 세밀하고 유연한 캐시 제어가 가능해요.</li>
    <li>`useOptimistic` 훅과 Server Actions를 결합하면, 서버 응답을 기다리지 않고 즉시 UI를 업데이트하는 Optimistic UI를 쉽게 구현하여 사용자 경험을 크게 향상시킬 수 있어요.</li>
  </ul>
</Blockquote>

이러한 기능들을 잘 활용하면 복잡한 데이터 동기화 로직을 줄이고, 더 빠르고 반응적인 웹 애플리케이션을 구축할 수 있게 돼요.
