## 🛡️ 타입 가드, 왜 중요할까요?
### 0️⃣ 런타임 타입 안전성 확보의 핵심
TypeScript는 컴파일 시점에 타입을 검사하여 오류를 미리 잡아주는 강력한 언어예요.<br />
하지만 자바스크립트로 컴파일된 후에는 타입 정보가 사라지기 때문에, 런타임에는 여전히 '타입 안전성' 문제가 발생할 수 있어요.<br />
예를 들어, 서버에서 받아온 데이터나 사용자 입력값은 TypeScript가 컴파일 시점에 알 수 없는 형태를 가질 수 있잖아요?<br />
이런 상황에서 특정 변수의 "실제" 타입을 런타임에 확인하고, 그 타입에 따라 안전하게 코드를 실행할 수 있도록 돕는 것이 바로 '타입 가드'예요.<br />
타입 가드를 사용하면 `any` 타입을 남용하지 않고도 유연하면서도 견고한 코드를 작성할 수 있답니다.

### 1️⃣ 타입 좁히기(Type Narrowing)의 이해
타입 가드의 핵심 개념은 바로 '타입 좁히기(Type Narrowing)'예요.<br />
TypeScript 컴파일러는 코드의 흐름을 분석하여 특정 시점에서 변수가 가질 수 있는 타입을 더 좁은 범위로 추론해요.<br />
예를 들어, `if (typeof value === 'string')`과 같은 조건문 안에서는 `value`가 `string` 타입임을 확신할 수 있게 되죠.<br />
이렇게 특정 조건문이나 연산자를 통해 변수의 타입을 더 구체적으로 만드는 과정을 타입 좁히기라고 부르고요.<br />
타입 가드는 이러한 타입 좁히기를 가능하게 하는 다양한 방법들을 통칭하는 용어라고 생각하시면 돼요.

## 🛠️ 기본적인 내장 타입 가드 살펴보기
TypeScript는 몇 가지 내장된 연산자를 통해 기본적인 타입 가드 기능을 제공하고 있어요.<br />
이들은 가장 흔하게 사용되며, 코드의 가독성과 안전성을 높이는 데 크게 기여한답니다.

### 0️⃣ `typeof` 연산자를 활용한 타입 가드
`typeof` 연산자는 원시 타입(primitive types)의 변수 타입을 확인하는 데 사용돼요.<br />
`'string'`, `'number'`, `'boolean'`, `'symbol'`, `'undefined'`, `'object'`, `'function'`, `'bigint'` 중 하나의 문자열을 반환하죠.<br />
특히 `string | number`와 같은 유니온 타입에서 특정 원시 타입으로 좁힐 때 매우 유용해요.

```tsx showLineNumbers {5-7, 9-11}
function printId(id: string | number) {
  if (typeof id === 'string') {
    // 이 블록 안에서 id는 string 타입으로 좁혀져요.
    console.log(id.toUpperCase()); // string 메서드 사용 가능
  } else {
    // 이 블록 안에서 id는 number 타입으로 좁혀져요.
    console.log(id.toFixed(2)); // number 메서드 사용 가능
  }
}

printId("hello"); // HELLO
printId(123.456); // 123.46
```

<Blockquote type="warning">
`typeof null`은 `'object'`를 반환한다는 점을 항상 기억해야 해요.<br />
`null` 체크가 필요하다면 `value === null`과 같이 명시적으로 비교해야 한답니다.
</Blockquote>

### 1️⃣ `instanceof` 연산자를 활용한 타입 가드
`instanceof` 연산자는 특정 객체가 특정 클래스의 인스턴스인지 확인할 때 사용해요.<br />
클래스 기반의 객체 지향 프로그래밍에서 매우 유용하게 활용될 수 있어요.

```tsx showLineNumbers {13-15, 17-19}
class Dog {
  bark() {
    console.log("멍멍!");
  }
}

class Cat {
  meow() {
    console.log("야옹!");
  }
}

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    // 이 블록 안에서 animal은 Dog 타입으로 좁혀져요.
    animal.bark();
  } else {
    // 이 블록 안에서 animal은 Cat 타입으로 좁혀져요.
    animal.meow();
  }
}

const myDog = new Dog();
const myCat = new Cat();

makeSound(myDog); // 멍멍!
makeSound(myCat); // 야옹!
```

### 2️⃣ `in` 연산자를 활용한 타입 가드
`in` 연산자는 특정 객체에 특정 속성(property)이 존재하는지 확인할 때 사용해요.<br />
주로 유니온 타입으로 묶인 객체들이 서로 다른 속성을 가질 때 유용하게 쓰여요.

```tsx showLineNumbers {12-14, 16-18}
interface Car {
  drive(): void;
  brand: string;
}

interface Boat {
  sail(): void;
  maxSpeed: number;
}

function operateVehicle(vehicle: Car | Boat) {
  if ('drive' in vehicle) {
    // 이 블록 안에서 vehicle은 Car 타입으로 좁혀져요.
    vehicle.drive();
    console.log(`브랜드: ${vehicle.brand}`);
  } else {
    // 이 블록 안에서 vehicle은 Boat 타입으로 좁혀져요.
    vehicle.sail();
    console.log(`최고 속도: ${vehicle.maxSpeed}노트`);
  }
}

const myCar: Car = { drive: () => console.log("운전해요!"), brand: "Tesla" };
const myBoat: Boat = { sail: () => console.log("항해해요!"), maxSpeed: 50 };

operateVehicle(myCar);
// 운전해요!
// 브랜드: Tesla

operateVehicle(myBoat);
// 항해해요!
// 최고 속도: 50노트
```

## 🧑‍💻 사용자 정의 타입 가드: 나만의 타입 가드 만들기
내장 타입 가드만으로는 복잡한 객체나 인터페이스의 타입을 좁히기 어려울 때가 많아요.<br />
이럴 때 '사용자 정의 타입 가드(User-Defined Type Guard)'를 사용하면 개발자가 직접 타입 좁히기 로직을 정의할 수 있어요.<br />
이는 `parameterName is Type`이라는 특별한 반환 타입을 가진 함수를 통해 구현된답니다.

### 0️⃣ `parameter is Type` 문법 이해하기
사용자 정의 타입 가드 함수는 다음과 같은 형태를 가져요.

```typescript
function isType(variable: any): variable is Type {
  // 여기에 variable이 Type인지 확인하는 로직을 작성해요.
  return typeof variable === 'string'; // 예시
}
```

여기서 `variable is Type`이 핵심이에요.<br />
이것은 함수가 `true`를 반환하면, TypeScript 컴파일러가 해당 `variable`이 `Type`으로 좁혀졌다고 인식하도록 지시하는 역할을 해요.<br />
함수 내부에서는 일반적인 자바스크립트 로직을 사용하여 런타임에 타입을 검사하고, 그 결과에 따라 `true` 또는 `false`를 반환하면 돼요.

### 1️⃣ 사용자 정의 타입 가드 구현 예시
여러 종류의 도형을 다루는 애플리케이션을 만든다고 가정해볼게요.<br />
`Circle`과 `Square` 인터페이스가 있고, 이들을 구분하는 타입 가드를 만들어볼 수 있어요.

```tsx showLineNumbers {10, 20-22, 24-26}
interface Circle {
  kind: "circle";
  radius: number;
}

interface Square {
  kind: "square";
  sideLength: number;
}

type Shape = Circle | Square;

// Circle 타입 가드 함수
function isCircle(shape: Shape): shape is Circle {
  return shape.kind === "circle";
}

// Square 타입 가드 함수
function isSquare(shape: Shape): shape is Square {
  return shape.kind === "square";
}

function getArea(shape: Shape): number {
  if (isCircle(shape)) {
    // 여기서는 shape가 Circle 타입임을 TypeScript가 알아요.
    return Math.PI * shape.radius ** 2;
  } else if (isSquare(shape)) {
    // 여기서는 shape가 Square 타입임을 TypeScript가 알아요.
    return shape.sideLength ** 2;
  }
  // 모든 가능한 경우를 처리했으므로, 여기에는 도달하지 않아요.
  // 만약 다른 타입이 추가된다면, 이곳에서 에러를 발생시키거나 기본값을 반환할 수 있어요.
  const _exhaustiveCheck: never = shape; // 모든 케이스를 처리했는지 컴파일러가 확인하도록 돕는 트릭
  return _exhaustiveCheck;
}

const myCircle: Shape = { kind: "circle", radius: 10 };
const mySquare: Shape = { kind: "square", sideLength: 5 };

console.log(`원의 넓이: ${getArea(myCircle)}`); // 원의 넓이: 314.1592653589793
console.log(`정사각형의 넓이: ${getArea(mySquare)}`); // 정사각형의 넓이: 25
```

<Blockquote type="tip">
위 예시에서 `kind` 속성과 같이 유니온 타입의 멤버를 구분하는 속성을 '식별자(discriminant)'라고 부르며, 이를 활용한 패턴을 '식별된 유니온(Discriminated Union)'이라고 해요.<br />
식별된 유니온은 사용자 정의 타입 가드를 더욱 효과적으로 사용할 수 있게 해주는 강력한 패턴이랍니다.
</Blockquote>

### 2️⃣ 비동기 로직에서의 타입 가드
비동기적으로 데이터를 받아올 때도 타입 가드는 유용하게 사용될 수 있어요.<br />
예를 들어, API 응답 데이터의 유효성을 검사할 때 활용할 수 있죠.

```tsx showLineNumbers {14-16, 25-27}
interface User {
  id: number;
  name: string;
  email: string;
}

// 사용자 정의 타입 가드: 객체가 User 인터페이스를 만족하는지 확인
function isUser(data: any): data is User {
  return (
    typeof data === 'object' &&
    data !== null &&
    typeof data.id === 'number' &&
    typeof data.name === 'string' &&
    typeof data.email === 'string'
  );
}

async function fetchUser(userId: number): Promise<User | null> {
  const response = await fetch(`https://api.example.com/users/${userId}`);
  const data = await response.json();

  if (isUser(data)) {
    // data가 User 타입으로 좁혀졌으므로 안전하게 접근 가능
    console.log(`사용자 이름: ${data.name}, 이메일: ${data.email}`);
    return data;
  } else {
    console.warn("API 응답이 User 타입과 일치하지 않아요.", data);
    return null;
  }
}

// 실제 API 호출 대신 목업 데이터로 테스트
// fetchUser(1).then(user => { /* ... */ });
```

<Blockquote type="info">
실제 프로덕션 환경에서는 `zod`, `yup` 등과 같은 스키마 유효성 검사 라이브러리를 사용하여 API 응답의 유효성을 훨씬 강력하게 검사하는 것이 일반적이에요.<br />
이러한 라이브러리들도 내부적으로 타입 가드와 유사한 방식으로 동작하며, 런타임 검사 후 TypeScript 타입 추론을 도와주는 기능을 제공한답니다.
</Blockquote>

## ⚠️ 타입 가드 사용 시 주의할 점
타입 가드는 매우 유용하지만, 몇 가지 주의할 점이 있어요.

### 0️⃣ 타입 가드의 정확성
사용자 정의 타입 가드는 개발자가 직접 로직을 작성하기 때문에, 그 정확성에 전적으로 의존해요.<br />
만약 타입 가드 로직이 잘못되었다면, TypeScript는 잘못된 타입 추론을 하게 되고, 런타임 오류로 이어질 수 있어요.<br />
따라서 타입 가드 함수는 항상 신중하게 작성하고, 필요한 경우 단위 테스트를 통해 검증하는 것이 좋아요.

### 1️⃣ 런타임에만 동작해요
TypeScript의 타입은 컴파일 시점에만 존재하고 런타임에는 사라진다는 점을 항상 기억해야 해요.<br />
타입 가드는 자바스크립트 코드로 컴파일된 후에도 런타임에 실제 값을 검사하는 로직으로 변환돼요.<br />
즉, 타입 가드는 "TypeScript의 타입 시스템을 돕기 위해 런타임 검사를 수행하는 자바스크립트 코드"라고 이해할 수 있어요.

## 📝 정리하며: 견고한 TypeScript 코드를 위한 필수 도구
오늘은 TypeScript의 타입 가드에 대해 깊이 있게 다뤄봤어요.<br />
`typeof`, `instanceof`, `in`과 같은 내장 타입 가드부터, 복잡한 타입을 안전하게 다룰 수 있게 해주는 사용자 정의 타입 가드까지, 다양한 활용법을 알아보았죠.

타입 가드는 단순히 오류를 피하는 것을 넘어, 코드의 가독성을 높이고 유지보수를 용이하게 하며, 궁극적으로는 더욱 견고하고 신뢰할 수 있는 애플리케이션을 만드는 데 필수적인 도구예요.<br />
초중급 개발자분들도 이제 타입 가드를 적극적으로 활용하여 TypeScript의 잠재력을 최대한 끌어내 보시길 바라요!
