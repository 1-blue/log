## 🤔 유틸리티 타입, 왜 필요할까요?

### 0️⃣ 복잡한 타입 정의의 간결화

TypeScript를 사용하다 보면, 기존에 정의된 타입의 일부 속성만 사용하거나, 특정 속성만 선택적으로 만들거나, 반대로 필수적으로 만들어야 하는 경우가 자주 발생해요.<br />
이때마다 새로운 타입을 일일이 수동으로 정의하려면 번거롭고 실수할 확률도 높아지겠죠? 유틸리티 타입은 이러한 반복적인 작업을 줄여주고, 훨씬 선언적이고 읽기 쉬운 코드를 작성할 수 있도록 도와줘요.

### 1️⃣ 코드 재사용성과 유지보수성 향상

유틸리티 타입을 사용하면 기존 타입을 재활용하여 다양한 파생 타입을 생성할 수 있어요. 이는 타입 정의의 중복을 줄이고, 원본 타입이 변경되었을 때 파생 타입들도 자동으로 업데이트되도록 하여 유지보수 비용을 절감하는 효과가 있어요.<br />
마치 함수를 만들어서 코드를 재사용하듯이, 타입을 재사용하는 것이라고 생각하시면 이해하기 쉬울 거예요.

## ⚙️ 핵심 유틸리티 타입 파헤치기

자주 사용되는 주요 유틸리티 타입들을 하나씩 살펴보면서 어떻게 활용할 수 있는지 알아볼게요.

### 1️⃣ `Partial<T>`: 모든 속성을 선택적으로 만들기

`Partial<T>`은 특정 타입 `T`의 모든 속성을 선택적(optional)으로 만들어주는 유틸리티 타입이에요.<br />
객체의 일부 속성만 업데이트하거나, 초기값을 설정할 때 유용하게 사용할 수 있어요.

```tsx showLineNumbers
interface User {
  id: number;
  name: string;
  email: string;
  age?: number; // age는 원래 선택적 속성
}

// User 타입의 모든 속성을 선택적으로 만듭니다.
type PartialUser = Partial<User>;

/*
PartialUser는 다음과 같습니다:
{
  id?: number;
  name?: string;
  email?: string;
  age?: number;
}
*/

const user1: User = { id: 1, name: '블루', email: 'blue@example.com' };

// PartialUser 타입은 모든 속성이 선택적이므로 일부만 있어도 유효합니다.
const partialUpdate: PartialUser = {
  name: '블루 개발자',
  age: 30,
};

// 부분 업데이트 함수 예시
function updateUser(user: User, updates: PartialUser): User {
  return { ...user, ...updates };
}

const updatedUser = updateUser(user1, partialUpdate);
console.log(updatedUser); // { id: 1, name: '블루 개발자', email: 'blue@example.com', age: 30 }
```

<Blockquote type="tip">
**실전 팁:** API 요청 시, 특정 리소스의 일부 필드만 업데이트해야 할 때 `Partial<T>`를 활용하면 유연하게 타입을 정의할 수 있어요. 예를 들어, `PATCH /users/{id}` 요청 바디에 `Partial<User>` 타입을 적용할 수 있어요.
</Blockquote>

### 2️⃣ `Required<T>`: 모든 속성을 필수로 만들기

`Required<T>`은 `Partial<T>`와 반대로, 특정 타입 `T`의 모든 속성을 필수(required)로 만들어주는 유틸리티 타입이에요.<br />
선택적 속성이 포함된 타입을 항상 모든 속성을 가진 상태로 처리해야 할 때 유용해요.

```tsx showLineNumbers
interface Product {
  id: string;
  name: string;
  price: number;
  description?: string; // description은 선택적 속성
}

// Product 타입의 모든 속성을 필수로 만듭니다.
type RequiredProduct = Required<Product>;

/*
RequiredProduct는 다음과 같습니다:
{
  id: string;
  name: string;
  price: number;
  description: string;
}
*/

const product1: RequiredProduct = {
  id: 'p001',
  name: '블루 키보드',
  price: 120000,
  description: '기계식 키보드입니다.', // description이 필수가 됨
};

// 에러 발생: 'description' 속성이 없으면 에러가 발생합니다.
// const product2: RequiredProduct = { id: 'p002', name: '블루 마우스', price: 50000 };
```

### 3️⃣ `Pick<T, K>`: 특정 속성만 선택하기

`Pick<T, K>`은 타입 `T`에서 `K`에 해당하는 속성들만 선택하여 새로운 타입을 만들어줘요.<br />
`K`는 `T`의 속성 이름들로 구성된 유니온 타입이어야 해요.

```tsx showLineNumbers
interface Order {
  orderId: string;
  userId: number;
  productId: string;
  quantity: number;
  orderDate: Date;
  status: 'pending' | 'completed' | 'cancelled';
}

// Order 타입에서 orderId, userId, orderDate 속성만 선택합니다.
type OrderSummary = Pick<Order, 'orderId' | 'userId' | 'orderDate'>;

/*
OrderSummary는 다음과 같습니다:
{
  orderId: string;
  userId: number;
  orderDate: Date;
}
*/

const summary: OrderSummary = {
  orderId: 'ORD-20260325-001',
  userId: 101,
  orderDate: new Date(),
};

// 에러 발생: status 속성은 OrderSummary에 없습니다.
// const invalidSummary: OrderSummary = { orderId: 'ORD-002', userId: 102, orderDate: new Date(), status: 'pending' };
```

<Blockquote type="info">
**활용 예시:** API 응답에서 필요한 데이터만 추출하여 DTO(Data Transfer Object) 타입을 정의할 때 `Pick<T, K>`을 사용하면 매우 효율적이에요.
</Blockquote>

### 4️⃣ `Omit<T, K>`: 특정 속성만 제외하기

`Omit<T, K>`은 `Pick<T, K>`과 반대로, 타입 `T`에서 `K`에 해당하는 속성들을 제외하고 새로운 타입을 만들어줘요.<br />
마찬가지로 `K`는 `T`의 속성 이름들로 구성된 유니온 타입이어야 해요.

```tsx showLineNumbers
interface UserProfile {
  id: number;
  username: string;
  email: string;
  passwordHash: string; // 보안상 클라이언트에 노출되면 안 되는 정보
  createdAt: Date;
  updatedAt: Date;
}

// UserProfile에서 passwordHash 속성을 제외합니다.
type PublicUserProfile = Omit<UserProfile, 'passwordHash'>;

/*
PublicUserProfile은 다음과 같습니다:
{
  id: number;
  username: string;
  email: string;
  createdAt: Date;
  updatedAt: Date;
}
*/

const publicProfile: PublicUserProfile = {
  id: 1,
  username: 'blue_dev',
  email: 'blue@dev.com',
  createdAt: new Date(),
  updatedAt: new Date(),
};

// 에러 발생: passwordHash는 PublicUserProfile에 없습니다.
// const invalidProfile: PublicUserProfile = { ...publicProfile, passwordHash: 'hashedpassword' };
```

<Blockquote type="warning">
**주의:** `Omit<T, K>`은 `Pick<T, Exclude<keyof T, K>>`와 동일하게 작동해요. 즉, `T`의 모든 키(`keyof T`)에서 `K`에 해당하는 키를 제외한 나머지를 선택하는 방식이에요.
</Blockquote>

### 5️⃣ `Readonly<T>`: 모든 속성을 읽기 전용으로 만들기

`Readonly<T>`은 타입 `T`의 모든 속성을 읽기 전용(readonly)으로 만들어줘요.<br />
불변성(immutability)을 강제해야 할 때 유용하게 사용할 수 있어요.

```tsx showLineNumbers
interface Configuration {
  apiUrl: string;
  timeout: number;
  debugMode: boolean;
}

// Configuration 타입의 모든 속성을 읽기 전용으로 만듭니다.
type ReadonlyConfig = Readonly<Configuration>;

/*
ReadonlyConfig는 다음과 같습니다:
{
  readonly apiUrl: string;
  readonly timeout: number;
  readonly debugMode: boolean;
}
*/

const appConfig: ReadonlyConfig = {
  apiUrl: 'https://api.example.com',
  timeout: 5000,
  debugMode: false,
};

// 에러 발생: 읽기 전용 속성이므로 값을 할당할 수 없습니다.
// appConfig.timeout = 10000;
```

## 🚀 고급 유틸리티 타입 활용법

이제 조금 더 복잡한 시나리오에서 빛을 발하는 유틸리티 타입들을 살펴볼게요.

### 1️⃣ `Exclude<T, U>`: 유니온 타입에서 특정 멤버 제외하기

`Exclude<T, U>`은 유니온 타입 `T`에서 `U`에 할당 가능한 타입들을 제외하여 새로운 유니온 타입을 생성해요.

```tsx showLineNumbers
type HttpMethod = 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH' | 'OPTIONS';

// HttpMethod에서 'GET'과 'POST'를 제외합니다.
type NonSafeHttpMethod = Exclude<HttpMethod, 'GET' | 'POST'>;

/*
NonSafeHttpMethod는 다음과 같습니다:
'PUT' | 'DELETE' | 'PATCH' | 'OPTIONS'
*/

const method1: NonSafeHttpMethod = 'DELETE';
// 에러 발생: 'GET'은 NonSafeHttpMethod에 포함되지 않습니다.
// const method2: NonSafeHttpMethod = 'GET';
```

### 2️⃣ `Extract<T, U>`: 유니온 타입에서 특정 멤버 추출하기

`Extract<T, U>`은 `Exclude<T, U>`와 반대로, 유니온 타입 `T`에서 `U`에 할당 가능한 타입들만 추출하여 새로운 유니온 타입을 생성해요.

```tsx showLineNumbers
type AllEvents = 'click' | 'hover' | 'submit' | 'scroll' | MouseEvent | KeyboardEvent;

// AllEvents에서 MouseEvent 또는 KeyboardEvent와 관련된 타입만 추출합니다.
type DomEvents = Extract<AllEvents, MouseEvent | KeyboardEvent>;

/*
DomEvents는 다음과 같습니다:
MouseEvent | KeyboardEvent
*/

let event1: DomEvents;
event1 = new MouseEvent('click');
// event1 = 'click'; // 에러 발생: 'click'은 DomEvents에 포함되지 않습니다.
```

### 3️⃣ `NonNullable<T>`: `null`과 `undefined` 제거하기

`NonNullable<T>`은 타입 `T`에서 `null`과 `undefined`를 제거하여 새로운 타입을 생성해요.<br />
값이 확실히 존재할 때의 타입을 정의할 때 유용해요.

```tsx showLineNumbers
type MaybeString = string | null | undefined;

// MaybeString에서 null과 undefined를 제거합니다.
type DefinitelyString = NonNullable<MaybeString>;

/*
DefinitelyString은 다음과 같습니다:
string
*/

const str1: DefinitelyString = 'hello';
// 에러 발생: null은 DefinitelyString에 할당할 수 없습니다.
// const str2: DefinitelyString = null;
```

## 🎯 함수 관련 유틸리티 타입

함수의 매개변수나 반환 값 타입을 추출하는 데 유용한 유틸리티 타입들도 있어요.

### 1️⃣ `Parameters<T>`: 함수의 매개변수 타입 추출

`Parameters<T>`은 함수 타입 `T`의 매개변수들을 튜플 타입으로 추출해줘요.

```tsx showLineNumbers
function greet(name: string, age: number): string {
  return `안녕하세요, ${name}님! ${age}살이시군요.`;
}

// greet 함수의 매개변수 타입을 추출합니다.
type GreetParams = Parameters<typeof greet>;

/*
GreetParams는 다음과 같습니다:
[name: string, age: number]
*/

const params: GreetParams = ['블루', 30];
// const invalidParams: GreetParams = ['블루']; // 에러 발생: age가 누락됨

console.log(greet(...params));
```

### 2️⃣ `ReturnType<T>`: 함수의 반환 타입 추출

`ReturnType<T>`은 함수 타입 `T`의 반환 타입을 추출해줘요.

```tsx showLineNumbers
function calculateSum(a: number, b: number): number {
  return a + b;
}

// calculateSum 함수의 반환 타입을 추출합니다.
type SumResult = ReturnType<typeof calculateSum>;

/*
SumResult는 다음과 같습니다:
number
*/

const result: SumResult = 100;
// const invalidResult: SumResult = '결과'; // 에러 발생: string은 number에 할당할 수 없음
```

### 3️⃣ `Awaited<T>`: Promise의 결과 타입 추출

`Awaited<T>`은 `Promise`의 결과 타입을 추출하거나, 중첩된 `Promise`를 재귀적으로 평탄화하여 최종 결과 타입을 얻을 때 사용해요.<br />
비동기 함수를 다룰 때 매우 유용해요.

```tsx showLineNumbers
async function fetchData(): Promise<{ data: string }> {
  return Promise.resolve({ data: 'Hello, Blue!' });
}

async function fetchNestedData(): Promise<Promise<number>> {
  return Promise.resolve(Promise.resolve(123));
}

// fetchData 함수의 반환 타입인 Promise<{ data: string }>에서 실제 데이터 타입을 추출합니다.
type DataResult = Awaited<ReturnType<typeof fetchData>>;

/*
DataResult는 다음과 같습니다:
{ data: string }
*/

const data: DataResult = { data: 'Success' };

// fetchNestedData 함수의 반환 타입인 Promise<Promise<number>>에서 최종 number 타입을 추출합니다.
type NestedDataResult = Awaited<ReturnType<typeof fetchNestedData>>;

/*
NestedDataResult는 다음과 같습니다:
number
*/

const nestedData: NestedDataResult = 456;
```

## ✨ 제네릭을 활용한 커스텀 유틸리티 타입 만들기

TypeScript의 내장 유틸리티 타입만으로는 모든 상황을 커버하기 어려울 때가 있어요.<br />
이럴 때는 제네릭(Generics)과 조건부 타입(Conditional Types)을 활용하여 직접 커스텀 유틸리티 타입을 만들 수 있어요.

### 0️⃣ `DeepPartial<T>`: 중첩된 객체까지 선택적으로 만들기

내장 `Partial<T>`은 최상위 속성만 선택적으로 만들어요.<br />
중첩된 객체의 모든 속성까지 재귀적으로 선택적으로 만들고 싶다면 `DeepPartial<T>`과 같은 커스텀 유틸리티 타입이 필요해요.

```tsx showLineNumbers {1-7}
type DeepPartial<T> = T extends object ? {
  [P in keyof T]?: DeepPartial<T[P]>;
} : T;

interface Settings {
  theme: 'dark' | 'light';
  notifications: {
    email: boolean;
    sms: boolean;
  };
  profile: {
    firstName: string;
    lastName: string;
    avatarUrl?: string;
  };
}

// DeepPartial을 적용하여 모든 속성과 중첩된 속성까지 선택적으로 만듭니다.
type PartialSettings = DeepPartial<Settings>;

/*
PartialSettings는 다음과 같습니다:
{
  theme?: "dark" | "light";
  notifications?: {
    email?: boolean;
    sms?: boolean;
  };
  profile?: {
    firstName?: string;
    lastName?: string;
    avatarUrl?: string;
  };
}
*/

const userSettings: PartialSettings = {
  notifications: {
    email: false, // 중첩된 속성도 선택적
  },
  profile: {
    firstName: '블루',
  },
};

console.log(userSettings);
```

<Blockquote type="tip">
**DeepPartial 해설:**<br />
`T extends object ? ... : T`는 조건부 타입이에요.<br />
만약 `T`가 객체 타입이라면 `[P in keyof T]?: DeepPartial<T[P]>`를 적용하고, 그렇지 않다면 `T` 자체를 반환해요.<br />
`[P in keyof T]?: ...`는 매핑된 타입으로, `T`의 모든 키 `P`에 대해 속성을 선택적(`?`)으로 만들고, 해당 속성 값(`T[P]`)에 다시 `DeepPartial`을 재귀적으로 적용해요. 이렇게 하면 중첩된 객체까지 모든 속성이 선택적으로 변환돼요.
</Blockquote>

## 📝 정리 및 다음 액션

### 0️⃣ 핵심 요약

오늘은 TypeScript의 강력한 기능인 유틸리티 타입에 대해 깊이 있게 살펴보았어요.<br />

*   **`Partial<T>`**: 모든 속성을 선택적으로 만들 때 사용해요.
*   **`Required<T>`**: 모든 속성을 필수로 만들 때 사용해요.
*   **`Pick<T, K>`**: 특정 속성만 선택하여 새로운 타입을 만들 때 사용해요.
*   **`Omit<T, K>`**: 특정 속성을 제외하여 새로운 타입을 만들 때 사용해요.
*   **`Readonly<T>`**: 모든 속성을 읽기 전용으로 만들 때 사용해요.
*   **`Exclude<T, U>`**: 유니온 타입에서 특정 멤버를 제외할 때 사용해요.
*   **`Extract<T, U>`**: 유니온 타입에서 특정 멤버를 추출할 때 사용해요.
*   **`NonNullable<T>`**: `null`과 `undefined`를 제거할 때 사용해요.
*   **`Parameters<T>`**: 함수의 매개변수 타입을 튜플로 추출할 때 사용해요.
*   **`ReturnType<T>`**: 함수의 반환 타입을 추출할 때 사용해요.
*   **`Awaited<T>`**: Promise의 최종 결과 타입을 추출할 때 사용해요.

또한, 제네릭과 조건부 타입을 활용하여 `DeepPartial<T>`와 같은 커스텀 유틸리티 타입을 만드는 방법도 알아보았어요.<br />
유틸리티 타입은 코드의 재사용성을 높이고, 타입 정의를 간결하게 하며, 궁극적으로 더 견고하고 유지보수하기 쉬운 TypeScript 애플리케이션을 만드는 데 필수적인 도구예요.
